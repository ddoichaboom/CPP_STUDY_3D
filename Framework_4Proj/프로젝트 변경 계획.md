 Phase 3: Client 프로젝트 (EXE → DLL 전환)

 핵심 변경: 출력 형식을 DLL로 변경

 3-1. 프로젝트 설정

 - 구성 형식: 동적 라이브러리(.dll) ← 기존 EXE에서 변경
 - 전처리기: CLIENT_EXPORTS 정의 추가
 - 런타임 라이브러리: /MDd (Debug) / /MD (Release)
 - 출력 디렉토리: $(SolutionDir)Client\Bin\
 - 추가 포함 디렉토리: $(SolutionDir)EngineSDK\Inc\
 - 추가 라이브러리 디렉토리: $(SolutionDir)EngineSDK\Lib\
 - 추가 종속성: Engine.lib

 3-2. Client_Defines.h 수정

 기존:
 #pragma once
 #include <Windows.h>
 #include <process.h>

 namespace Client
 {
     static const unsigned int g_iWinSizeX = { 1280 };
     static const unsigned int g_iWinSizeY = { 720 };
     enum class LEVEL { STATIC, LOADING, LOGO, GAMEPLAY, END };
 }

 extern HWND g_hWnd;

 using namespace Client;

 변경:
 #pragma once
 #include <Windows.h>
 #include <process.h>

 namespace Client
 {
     enum class LEVEL { STATIC, LOADING, LOGO, GAMEPLAY, END };
 }

 // g_hWnd 제거 → CGameInstance를 통해 접근
 // g_iWinSizeX, g_iWinSizeY 제거 → ENGINE_DESC에서 전달

 using namespace Client;

 핵심: extern HWND g_hWnd, g_iWinSizeX, g_iWinSizeY 전역 변수를 제거한다.

 3-3. CGameInstance에 HWND 접근자 추가 (Engine 측)

 전역 변수 제거의 대안으로, CGameInstance가 ENGINE_DESC의 hWnd를 저장하고 제공한다.

 Engine/Public/GameInstance.h에 추가:
 // ENGINE 영역에 추가
 HWND Get_hWnd() const { return m_hWnd; }
 _uint Get_WinSizeX() const { return m_iWinSizeX; }
 _uint Get_WinSizeY() const { return m_iWinSizeY; }

 Engine/Private/GameInstance.cpp의 Initialize_Engine()에서:
 m_hWnd = EngineDesc.hWnd;
 m_iWinSizeX = EngineDesc.iViewportWidth;
 m_iWinSizeY = EngineDesc.iViewportHeight;

 3-4. MainApp.h 수정 — CLIENT_DLL export 추가

 기존:
 class CMainApp final : public CBase

 변경:
 class CLIENT_DLL CMainApp final : public CBase

 3-5. MainApp.cpp 수정 — 전역 변수 참조 제거

 기존 Initialize():
 EngineDesc.hWnd = g_hWnd;            // 전역 변수 참조
 EngineDesc.iViewportWidth = g_iWinSizeX;
 EngineDesc.iViewportHeight = g_iWinSizeY;

 변경 — HWND를 매개변수로 받도록:
 // MainApp.h
 HRESULT Initialize(HWND hWnd, _uint iWinSizeX, _uint iWinSizeY);

 // Create도 매개변수 전달
 static CMainApp* Create(HWND hWnd, _uint iWinSizeX, _uint iWinSizeY);

 // MainApp.cpp Initialize()
 HRESULT CMainApp::Initialize(HWND hWnd, _uint iWinSizeX, _uint iWinSizeY)
 {
     ENGINE_DESC EngineDesc{};
     EngineDesc.hWnd = hWnd;
     EngineDesc.eWinMode = WINMODE::WIN;
     EngineDesc.iViewportWidth = iWinSizeX;
     EngineDesc.iViewportHeight = iWinSizeY;
     EngineDesc.iNumLevels = ETOUI(LEVEL::END);

     if (FAILED(m_pGameInstance->Initialize_Engine(EngineDesc, &m_pDevice, &m_pContext)))
         return E_FAIL;

     if (FAILED(Start_Level(LEVEL::LOGO)))
         return E_FAIL;

     return S_OK;
 }

 Phase 4: GameApp 프로젝트 (신규, EXE)

 4-1. 프로젝트 설정

 - 구성 형식: 애플리케이션(.exe)
 - 전처리기: 특별한 정의 없음
 - 런타임 라이브러리: /MDd (Debug) / /MD (Release)
 - 출력 디렉토리: $(SolutionDir)GameApp\Bin\
 - 추가 포함 디렉토리: $(SolutionDir)EngineSDK\Inc\;$(SolutionDir)ClientSDK\Inc\
 - 추가 라이브러리 디렉토리: $(SolutionDir)EngineSDK\Lib\;$(SolutionDir)ClientSDK\Lib\
 - 추가 종속성: Engine.lib, Client.lib
 - 하위 시스템: 창(/SUBSYSTEM:WINDOWS)

 4-2. GameApp.cpp — 기존 Client.cpp에서 이동

 기존 Client/Default/Client.cpp의 내용을 가져오되, 다음을 수정한다:

 #include "GameInstance.h"   // Engine SDK
 #include "MainApp.h"        // Client SDK

 // 윈도우 크기 상수 (GameApp 자체에서 정의)
 static const unsigned int g_iWinSizeX = 1280;
 static const unsigned int g_iWinSizeY = 720;

 // 전역 변수
 HINSTANCE   hInst;
 HWND        g_hWnd;

 // wWinMain
 int APIENTRY wWinMain(...)
 {
     // ... 윈도우 생성 (기존과 동일) ...

     // CMainApp 생성 — HWND와 윈도우 크기를 매개변수로 전달
     CMainApp* pMainApp = CMainApp::Create(g_hWnd, g_iWinSizeX, g_iWinSizeY);
     if (nullptr == pMainApp)
         return FALSE;

     CGameInstance* pGameInstance = CGameInstance::GetInstance();
     Safe_AddRef(pGameInstance);

     // Timer 등록
     pGameInstance->Add_Timer(TEXT("Timer_Default"));
     pGameInstance->Add_Timer(TEXT("Timer_60"));

     // 메시지 루프 (기존과 동일)
     _float fTimeAcc = {};
     while (true)
     {
         if (PeekMessage(&msg, nullptr, 0, 0, PM_REMOVE))
         {
             if (WM_QUIT == msg.message) break;
             TranslateMessage(&msg);
             DispatchMessageW(&msg);
         }

         pGameInstance->Compute_Timer(TEXT("Timer_Default"));
         fTimeAcc += pGameInstance->Get_TimeDelta(TEXT("Timer_Default"));

         if (fTimeAcc >= 1.f / 60.f)
         {
             pGameInstance->Compute_Timer(TEXT("Timer_60"));
             pMainApp->Update(pGameInstance->Get_TimeDelta(TEXT("Timer_60")));
             pMainApp->Render();
             fTimeAcc = 0.f;
         }
     }

     Safe_Release(pGameInstance);
     Safe_Release(pMainApp);
     return (int)msg.wParam;
 }

 Phase 5: UpdateLib.bat 확장

 기존 Engine 헤더/라이브러리 복사에 Client도 추가한다.

 :: Engine SDK
 copy Engine\Public\*.h EngineSDK\Inc\
 copy Engine\Bin\Engine.lib EngineSDK\Lib\

 :: Client SDK (신규)
 copy Client\Public\*.h ClientSDK\Inc\
 copy Client\Bin\Client.lib ClientSDK\Lib\

 주의: Client의 Private 헤더(Level_Logo.h, Loader.h 등)는 복사하지 않는다.
 Export되는 클래스의 헤더(MainApp.h, BackGround.h, Client_Defines.h)만 ClientSDK에 복사한다.

 → 이를 위해 복사 대상을 개별 지정하거나, Public 폴더를 export용/internal용으로 분리할 수 있다.

 간단한 방법: Client/Public에는 export 대상 헤더만 두고, internal 헤더(Level_*.h, Loader.h)는 Client/Internal/ 같은 별도 폴더로 분리한다.

 Phase 6: CRT 통일 검증

 모든 프로젝트에서 반드시 확인할 설정:

 ┌──────────┬───────────┬───────────────────────────┬─────────────────────────────┐
 │ 프로젝트 │ 구성 형식 │ 런타임 라이브러리 (Debug) │ 런타임 라이브러리 (Release) │
 ├──────────┼───────────┼───────────────────────────┼─────────────────────────────┤
 │ Engine   │ DLL       │ /MDd                      │ /MD                         │
 ├──────────┼───────────┼───────────────────────────┼─────────────────────────────┤
 │ Client   │ DLL       │ /MDd                      │ /MD                         │
 ├──────────┼───────────┼───────────────────────────┼─────────────────────────────┤
 │ GameApp  │ EXE       │ /MDd                      │ /MD                         │
 └──────────┴───────────┴───────────────────────────┴─────────────────────────────┘

 3개 프로젝트 모두 **동적 CRT (/MD 계열)**로 통일해야 한다.
 하나라도 /MT(정적 CRT)를 사용하면 DLL 경계에서 delete this 시 힙 충돌 발생.
