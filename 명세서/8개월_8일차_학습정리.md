## 1. 이론: Phong 라이팅 모델

### 1.1 조명의 3요소

실시간 3D 렌더링에서 Phong 라이팅은 빛을 세 가지 성분으로 분해하여 계산한다:

```
최종 색상 = Diffuse(난반사) + Ambient(환경광) + Specular(정반사)
```

| 성분 | 물리적 의미 | 시각적 효과 |
|------|-------------|-------------|
| **Diffuse** | 빛이 표면에 닿아 모든 방향으로 고르게 반사 | 표면의 밝기(빛을 향한 면은 밝고, 등진 면은 어둡다) |
| **Ambient** | 직접 빛이 닿지 않는 곳에도 간접광으로 인한 최소 밝기 | 그림자 영역이 완전히 검게 되지 않도록 하는 바닥값 |
| **Specular** | 빛이 매끈한 표면에서 특정 방향으로 강하게 반사 | 하이라이트(반짝이는 점), 시점에 따라 위치 변함 |

### 1.2 Lambert Diffuse (난반사)

표면이 빛을 향할수록 밝다. 빛 방향 벡터와 표면 노멀의 내적으로 계산한다.

```
Diffuse = max(dot(-LightDir, Normal), 0)
```

- `LightDir`: 빛이 밖에서 **표면으로 들어오는** 방향 (예: (1, -1, 1)). `-1`을 곱하면 표면에서 밖으로 나가는 방향이 되어 법선 벡터와 같은 기준점에서 출발하므로 내적이 성립한다.
- `Normal`: 표면의 법선 벡터 (월드 공간으로 변환 필요)
- `max(..., 0)`: 내적이 음수(빛의 반대편)이면 0으로 클램핑

### 1.3 Ambient (환경광)

모든 픽셀에 일정한 최소 밝기를 더한다. 물리적으로는 간접광(벽에 반사된 빛 등)을 단순화한 것.

```
Ambient = LightAmbient * MaterialAmbient
```

- `LightAmbient`: 광원의 환경광 색상 (보통 흰색 = (1,1,1,1))
- `MaterialAmbient`: 재질의 환경광 반응도 (예: (0.4, 0.4, 0.4, 1) → 최소 40% 밝기 보장)

### 1.4 Phong Specular (정반사)

빛이 표면에 반사된 방향과 카메라가 바라보는 방향이 가까울수록 밝은 하이라이트가 생긴다.

```
Reflect   = reflect(LightDir, Normal)          // 반사 벡터
ViewDir   = normalize(WorldPos - CamPosition)  // 시선 벡터
Specular  = pow(max(dot(-ViewDir, Reflect), 0), Shininess)
```

- `reflect(I, N)`: HLSL 내장 함수. 입사 벡터 `I`를 노멀 `N` 기준으로 반사
- `Shininess` (= 50.f): 지수가 클수록 하이라이트가 작고 날카로움. 작으면 넓고 부드러움
- `ViewDir = WorldPos - CamPos`: 카메라에서 표면을 향하는 방향. `-ViewDir`은 표면에서 카메라를 향하는 방향이 되어, 표면에서 나가는 반사 벡터와 같은 기준점에서 출발하므로 내적이 성립한다.

### 1.5 최종 색상 조합

```
FinalColor = LightDiffuse * MtrlDiffuse * saturate(Diffuse + Ambient)
           + (LightSpecular * MtrlSpecular) * Specular
```

- `MtrlDiffuse`: 텍스처 샘플링 결과 (표면의 고유 색상)
- `saturate()`: [0, 1] 범위로 클램핑
- Diffuse와 Ambient를 합친 값에 `saturate`를 걸어 1.0을 넘지 않도록 제한
- Specular는 별도로 더한다 (하이라이트는 텍스처 색과 무관하게 빛 색상 그대로)

### 1.6 노멀 변환 — 왜 w = 0인가

```hlsl
Out.vNormal = normalize(mul(float4(In.vNormal, 0.f), g_WorldMatrix));
```

- Position은 `w = 1.f` → **점(Point)**: 이동(Translation)의 영향을 받음
- Normal은 `w = 0.f` → **방향(Direction)**: 이동의 영향을 받지 않고 회전/스케일만 적용
- 노멀을 월드 공간으로 변환해야 라이팅 계산이 월드 공간의 빛 방향과 일치

### 1.7 월드 좌표가 필요한 이유

```hlsl
Out.vWorldPos = mul(float4(In.vPosition, 1.f), g_WorldMatrix);
```

Specular 계산에 `시선 벡터 = WorldPos - CamPosition`이 필요하다.
클립/스크린 공간의 좌표가 아닌 **월드 공간 좌표**여야 카메라 월드 좌표와 빼기 연산이 의미를 갖는다.

## 2. CShader::Bind_RawValue — 새 엔진 메서드

### 2.1 필요한 이유

기존 `Bind_Matrix`는 `_float4x4` 행렬만, `Bind_SRV`는 텍스처 SRV만 바인딩할 수 있다.
라이팅에는 `_float4` 벡터(빛 방향, 색상, 카메라 위치 등)를 셰이더에 전달해야 하므로, **임의 크기의 raw 데이터**를 바인딩하는 범용 메서드가 필요하다.

### 2.2 Shader.h 변경

```cpp
// === Shader.h에 추가 ===
public:
    HRESULT Bind_Matrix(const _char* pConstantName, const _float4x4* pMatrix);
    HRESULT Bind_SRV(const _char* pConstantName, ID3D11ShaderResourceView* pSRV);
    HRESULT Bind_RawValue(const _char* pConstantName, const void* pValue, _uint iLength);  // NEW
```

### 2.3 Shader.cpp 구현

```cpp
HRESULT CShader::Bind_RawValue(const _char* pConstantName, const void* pValue, _uint iLength)
{
    // (1) Effect에서 이름으로 변수 검색
    ID3DX11EffectVariable* pVariable = m_pEffect->GetVariableByName(pConstantName);
    if (!pVariable->IsValid())
        return E_FAIL;

    // (2) SetRawValue로 바이트 단위 데이터 전송
    return pVariable->SetRawValue(pValue, 0, iLength);
}
```

**핵심 포인트:**
- `GetVariableByName`으로 HLSL 변수를 찾는 것은 `Bind_Matrix`/`Bind_SRV`와 동일
- 차이점: `AsMatrix()`나 `AsShaderResource()`로 캐스팅하지 않고, `ID3DX11EffectVariable`의 `SetRawValue()`를 직접 호출
- `SetRawValue(pValue, 0, iLength)`: `pValue`에서 `iLength` 바이트를 offset 0부터 복사
- `_float4` 바인딩 시: `sizeof(_float4)` = 16바이트

### 2.4 사용 예시

```cpp
// 카메라 위치를 셰이더에 전달
m_pShaderCom->Bind_RawValue("g_vCamPosition",
    m_pGameInstance->Get_CamPosition(), sizeof(_float4));
```

## 3. CVIBuffer_Terrain — 정점 노멀 계산

### 3.1 현재 문제

현재 `VIBuffer_Terrain.cpp`에서 노멀을 `(0, 0, 0)`으로 초기화한 뒤 아무 계산 없이 버텍스 버퍼를 생성한다.
노멀이 영벡터이면 `dot(LightDir, Normal)` = 0이 되어 Diffuse 라이팅이 전혀 동작하지 않는다.

### 3.2 핵심 알고리즘: 면 노멀 → 정점 노멀

1. 각 쿼드(사각형)를 2개의 삼각형으로 분할
2. 각 삼각형의 **면 노멀**(face normal)을 두 변의 크로스 프로덕트로 계산
3. 삼각형을 구성하는 3개 정점에 면 노멀을 **누적**(+=)
4. 모든 인덱스 처리 후, 전체 정점의 노멀을 **정규화**(normalize)

이 방법이 **Smooth Shading**을 만든다 — 하나의 정점이 여러 삼각형에 공유되므로, 면 노멀들의 평균이 정점 노멀이 된다.

### 3.3 버퍼 생성 순서 변경

**현재 (노멀 없음)**:
```
정점 데이터 생성 → VB 생성 → 인덱스 데이터 생성 → IB 생성
```

**변경 후 (노멀 있음)**:
```
정점 데이터 생성 → 인덱스 데이터 생성 (+ 노멀 계산) → 노멀 정규화 → VB 생성 → IB 생성
```

VB를 인덱스 처리 후에 생성해야 한다. 인덱스를 순회하면서 정점 노멀을 누적하므로, VB를 먼저 만들면 CPU 메모리의 정점 데이터가 삭제되어 노멀을 쓸 수 없다.

### 3.4 VIBuffer_Terrain.cpp 변경 — 전체 코드

```cpp
HRESULT CVIBuffer_Terrain::Initialize_Prototype(const _tchar* pHeightMapFilePath)
{
    // (1) BMP Height Map 읽기 — 기존과 동일
    _ulong          dwByte = { };
    HANDLE          hFile = CreateFile(pHeightMapFilePath, GENERIC_READ, 0, nullptr, OPEN_EXISTING, FILE_ATTRIBUTE_NORMAL, nullptr);
    if (0 == hFile)
        return E_FAIL;

    BITMAPFILEHEADER    fh{};
    BITMAPINFOHEADER    ih{};
    _uint* pPixels = { nullptr };

    ReadFile(hFile, &fh, sizeof fh, &dwByte, nullptr);
    ReadFile(hFile, &ih, sizeof ih, &dwByte, nullptr);
    m_iNumVerticesX = ih.biWidth;
    m_iNumVerticesZ = ih.biHeight;
    m_iNumVertices  = m_iNumVerticesX * m_iNumVerticesZ;

    pPixels = new _uint[m_iNumVertices];
    ZeroMemory(pPixels, sizeof(_uint) * m_iNumVertices);
    ReadFile(hFile, pPixels, sizeof(_uint) * m_iNumVertices, &dwByte, nullptr);
    CloseHandle(hFile);

    // (2) 버퍼 속성 설정 — 기존과 동일
    m_iNumVertexBuffers = 1;
    m_iVertexStride     = sizeof(VTXNORTEX);
    m_iNumIndices       = (m_iNumVerticesX - 1) * (m_iNumVerticesZ - 1) * 2 * 3;
    m_iIndexStride      = 4;
    m_eIndexFormat      = DXGI_FORMAT_R32_UINT;
    m_ePrimitiveType    = D3D11_PRIMITIVE_TOPOLOGY_TRIANGLELIST;

    // (3) 정점 데이터 생성 — 기존과 동일 (노멀은 (0,0,0)으로 초기화)
    D3D11_BUFFER_DESC   VertexBufferDesc{};
    VertexBufferDesc.ByteWidth           = m_iVertexStride * m_iNumVertices;
    VertexBufferDesc.Usage               = D3D11_USAGE_DEFAULT;
    VertexBufferDesc.BindFlags           = D3D11_BIND_VERTEX_BUFFER;
    VertexBufferDesc.CPUAccessFlags      = 0;
    VertexBufferDesc.MiscFlags           = 0;
    VertexBufferDesc.StructureByteStride = m_iVertexStride;

    VTXNORTEX* pVertices = new VTXNORTEX[m_iNumVertices];
    ZeroMemory(pVertices, sizeof(VTXNORTEX) * m_iNumVertices);

    for (size_t i = 0; i < m_iNumVerticesZ; i++)
    {
        for (size_t j = 0; j < m_iNumVerticesX; j++)
        {
            size_t iIndex = i * m_iNumVerticesX + j;
            pVertices[iIndex].vPosition = _float3(j, (pPixels[iIndex] & 0x000000ff) / 10.f, i);
            pVertices[iIndex].vNormal   = _float3(0.f, 0.f, 0.f);
            pVertices[iIndex].vTexcoord = _float2(j / (m_iNumVerticesX - 1.f), i / (m_iNumVerticesZ - 1.f));
        }
    }

    // ========== 여기서부터 변경 ==========
    // *** VB 생성을 인덱스 처리 후로 이동 ***

    // (4) 인덱스 데이터 생성 + 노멀 계산
    D3D11_BUFFER_DESC   IndexBufferDesc{};
    IndexBufferDesc.ByteWidth           = m_iIndexStride * m_iNumIndices;
    IndexBufferDesc.Usage               = D3D11_USAGE_DEFAULT;
    IndexBufferDesc.BindFlags           = D3D11_BIND_INDEX_BUFFER;
    IndexBufferDesc.CPUAccessFlags      = 0;
    IndexBufferDesc.MiscFlags           = 0;
    IndexBufferDesc.StructureByteStride = m_iIndexStride;

    _uint* pIndices = new _uint[m_iNumIndices];
    ZeroMemory(pIndices, sizeof(_uint) * m_iNumIndices);

    _uint iNumIndices = {};

    for (size_t i = 0; i < m_iNumVerticesZ - 1; i++)
    {
        for (size_t j = 0; j < m_iNumVerticesX - 1; j++)
        {
            size_t iIndex = i * m_iNumVerticesX + j;

            // 쿼드의 4개 정점 인덱스 (좌상 → 우상 → 우하 → 좌하)
            _uint iIndices[4] = {
                iIndex + m_iNumVerticesX,        // [0] 좌상
                iIndex + m_iNumVerticesX + 1,    // [1] 우상
                iIndex + 1,                      // [2] 우하
                iIndex                           // [3] 좌하
            };

            // ===== 첫 번째 삼각형 (좌상 - 우상 - 우하) =====
            _vector vSour, vDest, vNormal;

            pIndices[iNumIndices++] = iIndices[0];
            pIndices[iNumIndices++] = iIndices[1];
            pIndices[iNumIndices++] = iIndices[2];

            // 면 노멀 = Cross(edge1, edge2), 정규화
            vSour   = XMLoadFloat3(&pVertices[iIndices[1]].vPosition) - XMLoadFloat3(&pVertices[iIndices[0]].vPosition);
            vDest   = XMLoadFloat3(&pVertices[iIndices[2]].vPosition) - XMLoadFloat3(&pVertices[iIndices[1]].vPosition);
            vNormal = XMVector3Normalize(XMVector3Cross(vSour, vDest));

            // 삼각형을 구성하는 3개 정점에 면 노멀 누적
            XMStoreFloat3(&pVertices[iIndices[0]].vNormal, XMLoadFloat3(&pVertices[iIndices[0]].vNormal) + vNormal);
            XMStoreFloat3(&pVertices[iIndices[1]].vNormal, XMLoadFloat3(&pVertices[iIndices[1]].vNormal) + vNormal);
            XMStoreFloat3(&pVertices[iIndices[2]].vNormal, XMLoadFloat3(&pVertices[iIndices[2]].vNormal) + vNormal);

            // ===== 두 번째 삼각형 (좌상 - 우하 - 좌하) =====
            pIndices[iNumIndices++] = iIndices[0];
            pIndices[iNumIndices++] = iIndices[2];
            pIndices[iNumIndices++] = iIndices[3];

            vSour   = XMLoadFloat3(&pVertices[iIndices[2]].vPosition) - XMLoadFloat3(&pVertices[iIndices[0]].vPosition);
            vDest   = XMLoadFloat3(&pVertices[iIndices[3]].vPosition) - XMLoadFloat3(&pVertices[iIndices[2]].vPosition);
            vNormal = XMVector3Normalize(XMVector3Cross(vSour, vDest));

            XMStoreFloat3(&pVertices[iIndices[0]].vNormal, XMLoadFloat3(&pVertices[iIndices[0]].vNormal) + vNormal);
            XMStoreFloat3(&pVertices[iIndices[2]].vNormal, XMLoadFloat3(&pVertices[iIndices[2]].vNormal) + vNormal);
            XMStoreFloat3(&pVertices[iIndices[3]].vNormal, XMLoadFloat3(&pVertices[iIndices[3]].vNormal) + vNormal);
        }
    }

    // (5) 모든 정점의 노멀 정규화 (누적된 면 노멀들의 평균 방향)
    for (size_t i = 0; i < m_iNumVertices; i++)
        XMStoreFloat3(&pVertices[i].vNormal, XMVector3Normalize(XMLoadFloat3(&pVertices[i].vNormal)));

    // (6) VB 생성 (노멀 계산 완료 후)
    D3D11_SUBRESOURCE_DATA  VertexInitialData{};
    VertexInitialData.pSysMem = pVertices;

    if (FAILED(m_pDevice->CreateBuffer(&VertexBufferDesc, &VertexInitialData, &m_pVB)))
        return E_FAIL;

    // (7) IB 생성
    D3D11_SUBRESOURCE_DATA  IndexInitialData{};
    IndexInitialData.pSysMem = pIndices;

    if (FAILED(m_pDevice->CreateBuffer(&IndexBufferDesc, &IndexInitialData, &m_pIB)))
        return E_FAIL;

    Safe_Delete_Array(pVertices);
    Safe_Delete_Array(pIndices);
    Safe_Delete_Array(pPixels);

    return S_OK;
}
```

### 3.5 크로스 프로덕트와 방향

```
        [0] 좌상 ──── [1] 우상
         │  ╲  △1  │
         │   ╲     │
         │ △2  ╲   │
        [3] 좌하 ──── [2] 우하
```

- **△1** (0→1→2): `edge1 = V1 - V0`, `edge2 = V2 - V1`, `Normal = Cross(edge1, edge2)`
- **△2** (0→2→3): `edge1 = V2 - V0`, `edge2 = V3 - V2`, `Normal = Cross(edge1, edge2)`
- DirectX는 **좌수 좌표계(Left-Handed)** + **시계 방향(CW) 와인딩** → Cross 결과가 위(+Y)를 향함

## 4. Shader_VtxNorTex.hlsl — Phong 라이팅 구현

### 4.1 현재 vs 변경 후 비교

| 항목 | 현재 (7일차) | 변경 후 (8일차) |
|------|-------------|----------------|
| 텍스처 변수명 | `g_Texture` | `g_DiffuseTexture` |
| 카메라 위치 | 없음 | `vector g_vCamPosition` |
| 광원 변수 | 없음 | `g_vLightDir`, `g_vLightDiffuse`, `g_vLightAmbient`, `g_vLightSpecular` |
| 재질 변수 | 없음 | `g_vMtrlAmbient`, `g_vMtrlSpecular` |
| VS_OUT | `vPosition`, `vTexcoord` | + `vNormal`(NORMAL), `vWorldPos`(TEXCOORD1) |
| VS 로직 | WVP 변환만 | + 노멀 월드 변환, 월드 좌표 전달 |
| PS 로직 | 텍스처 샘플 + 알파 테스트 | + Diffuse + Ambient + Specular 라이팅 |

### 4.2 전체 HLSL 코드

```hlsl
/* ===== 전역 변수 ===== */
float4x4 g_WorldMatrix, g_ViewMatrix, g_ProjMatrix;

vector g_vCamPosition;      // 카메라 월드 좌표 (Specular 계산용)

/* 광원 (Light) */
vector g_vLightDir     = vector(1.f, -1.f, 1.f, 0.f);    // 빛이 표면으로 들어오는 방향 (방향광)
vector g_vLightDiffuse = vector(1.f, 1.f, 1.f, 1.f);     // 빛의 난반사 색상
vector g_vLightAmbient = vector(1.f, 1.f, 1.f, 1.f);     // 빛의 환경광 색상
vector g_vLightSpecular= vector(1.f, 1.f, 1.f, 1.f);     // 빛의 정반사 색상

/* 재질 (Material) */
texture2D g_DiffuseTexture;                                // 텍스처 (재질의 난반사 색상)
vector g_vMtrlAmbient  = vector(0.4f, 0.4f, 0.4f, 1.f);  // 재질의 환경광 반응도 (최소 40%)
vector g_vMtrlSpecular = vector(1.f, 1.f, 1.f, 1.f);     // 재질의 정반사 반응도

sampler DefaultSampler = sampler_state
{
    Filter   = MIN_MAG_MIP_LINEAR;
    AddressU = wrap;
    AddressV = wrap;
};

/* ===== 정점 셰이더 ===== */
struct VS_IN
{
    float3 vPosition : POSITION;
    float3 vNormal   : NORMAL;
    float2 vTexcoord : TEXCOORD0;
};

struct VS_OUT
{
    float4 vPosition : SV_POSITION;
    float4 vNormal   : NORMAL;         // NEW: 월드 공간 노멀
    float2 vTexcoord : TEXCOORD0;
    float4 vWorldPos : TEXCOORD1;      // NEW: 월드 공간 위치
};

VS_OUT VS_MAIN(VS_IN In)
{
    VS_OUT Out;

    // WVP 변환 (기존과 동일)
    float4 vPosition = mul(float4(In.vPosition, 1.f), g_WorldMatrix);
    vPosition = mul(vPosition, g_ViewMatrix);
    vPosition = mul(vPosition, g_ProjMatrix);

    Out.vPosition = vPosition;

    // NEW: 노멀을 월드 공간으로 변환 (w=0 → 이동 무시, 회전/스케일만 적용)
    Out.vNormal   = normalize(mul(float4(In.vNormal, 0.f), g_WorldMatrix));

    Out.vTexcoord = In.vTexcoord;

    // NEW: 월드 좌표 전달 (Specular 시선 벡터 계산용)
    Out.vWorldPos = mul(float4(In.vPosition, 1.f), g_WorldMatrix);

    return Out;
}

/* ===== 픽셀 셰이더 ===== */
struct PS_IN
{
    float4 vPosition : SV_POSITION;
    float4 vNormal   : NORMAL;
    float2 vTexcoord : TEXCOORD0;
    float4 vWorldPos : TEXCOORD1;
};

struct PS_OUT
{
    float4 vColor : SV_TARGET0;
};

PS_OUT PS_MAIN(PS_IN In)
{
    PS_OUT Out;

    // (1) 텍스처 샘플링 (재질의 Diffuse 색상)
    vector vMtrlDiffuse = g_DiffuseTexture.Sample(DefaultSampler, In.vTexcoord);
    if (vMtrlDiffuse.a < 0.1f)
        discard;

    // (2) Diffuse (Lambert) + Ambient
    //     dot(-LightDir, Normal): -LightDir은 표면에서 밖으로 나가는 방향, Normal과 같은 기준점에서 내적
    //     + Ambient: 최소 밝기 보장
    vector vShade = max(dot(normalize(g_vLightDir) * -1.f, In.vNormal), 0.f)
                    + (g_vLightAmbient * g_vMtrlAmbient);

    // (3) Specular (Phong Reflection)
    //     빛의 반사 벡터와 시선 벡터가 가까울수록 밝은 하이라이트
    vector vReflect = reflect(normalize(g_vLightDir), normalize(In.vNormal));
    vector vLook    = In.vWorldPos - g_vCamPosition;

    float fSpecular = pow(max(dot(normalize(vLook) * -1.f, normalize(vReflect)), 0.f), 50.f);

    // (4) 최종 색상 조합
    Out.vColor = g_vLightDiffuse * vMtrlDiffuse * saturate(vShade)
               + (g_vLightSpecular * g_vMtrlSpecular) * fSpecular;

    return Out;
}

technique11 DefaultTechnique
{
    pass DefaultPass
    {
        VertexShader = compile vs_5_0 VS_MAIN();
        PixelShader  = compile ps_5_0 PS_MAIN();
    }
}
```

### 4.3 HLSL 내장 함수 정리

| 함수 | 설명 | 사용처 |
|------|------|--------|
| `normalize(v)` | 벡터 정규화 (길이 1로) | 빛 방향, 노멀, 시선 벡터 |
| `dot(a, b)` | 내적 (두 벡터의 방향 유사도, -1~1) | Diffuse, Specular |
| `max(a, b)` | 큰 값 반환 | 음수 클램핑 (0 이하 → 0) |
| `reflect(I, N)` | 입사 벡터 I를 노멀 N 기준으로 반사 | Specular 반사 벡터 |
| `pow(base, exp)` | 거듭제곱 | Specular 지수 (Shininess) |
| `saturate(v)` | [0, 1] 범위 클램핑 | 최종 셰이드 값 |

### 4.4 텍스처 변수명 변경 이유

`g_Texture` → `g_DiffuseTexture`

라이팅이 도입되면서 텍스처의 역할이 명확해짐 — "재질의 난반사(Diffuse) 색상"을 담당하는 텍스처.
향후 NormalMap, SpecularMap 등 추가 텍스처가 생길 수 있으므로 용도를 구분하는 네이밍.

> **주의:** HLSL 변수명이 바뀌었으므로 C++ 쪽의 `Bind_ShaderResource` 호출도 변경 필요.

## 5. Terrain.cpp — C++ 바인딩 변경

### 5.1 Bind_ShaderResources 변경

```cpp
HRESULT CTerrain::Bind_ShaderResources()
{
    // World 행렬 (기존과 동일)
    if (FAILED(m_pTransformCom->Bind_ShaderResource(m_pShaderCom, "g_WorldMatrix")))
        return E_FAIL;

    // View/Proj (기존과 동일)
    if (FAILED(m_pShaderCom->Bind_Matrix("g_ViewMatrix", m_pGameInstance->Get_Transform(D3DTS::VIEW))))
        return E_FAIL;
    if (FAILED(m_pShaderCom->Bind_Matrix("g_ProjMatrix", m_pGameInstance->Get_Transform(D3DTS::PROJ))))
        return E_FAIL;

    // ===== 변경: 텍스처 변수명 "g_Texture" → "g_DiffuseTexture" =====
    if (FAILED(m_pTextureCom->Bind_ShaderResource(m_pShaderCom, "g_DiffuseTexture", 0)))
        return E_FAIL;

    // ===== NEW: 카메라 위치 바인딩 (Specular 계산용) =====
    if (FAILED(m_pShaderCom->Bind_RawValue("g_vCamPosition", m_pGameInstance->Get_CamPosition(), sizeof(_float4))))
        return E_FAIL;

    return S_OK;
}
```

### 5.2 라이팅 파라미터는 어디서 바인딩하는가?

8일차 수업 코드에서 `g_vLightDir`, `g_vLightDiffuse` 등은 HLSL에 **기본값이 지정**되어 있다:

```hlsl
vector g_vLightDir = vector(1.f, -1.f, 1.f, 0.f);
```

C++에서 별도로 바인딩하지 않으면 이 기본값이 사용된다.
향후 동적 라이팅(여러 광원, 시간에 따른 변화 등)을 구현하려면 C++에서 `Bind_RawValue`로 런타임에 값을 전달하면 된다:

```cpp
// 향후 확장 예시 — 현재 수업에서는 불필요
_float4 vLightDir = { 1.f, -1.f, 1.f, 0.f };
m_pShaderCom->Bind_RawValue("g_vLightDir", &vLightDir, sizeof(_float4));
```
