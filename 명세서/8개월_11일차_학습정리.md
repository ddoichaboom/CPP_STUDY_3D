# 8개월차 11일차 - Material 시스템, PreTransformMatrix, 메시별 렌더링

## 개요

10일차에서는 Assimp를 이용해 FBX 모델을 로딩하고 CMesh/CModel 구조로 GPU 버퍼를 생성하여 화면에 그리는 데까지 도달했다. 그러나 **모델에 내장된 텍스처(재질)를 추출하지 못해** 셰이더에서 텍스처 대신 고정값(1.f)을 사용했고, 모델의 스케일/방향을 런타임에서만 Transform으로 조정해야 했다.

11일차에서는 이 두 가지 한계를 해결한다:

- **CMaterial** 신규 클래스 — FBX 내장 텍스처(Diffuse, Normal, Specular 등)를 Assimp의 aiMaterial에서 추출하여 SRV로 로딩
- **PreTransformMatrix** — 모델 로딩 시점에 스케일/회전을 정점 데이터에 직접 적용 (Bake)
- **메시별 렌더링** — CModel::Render()가 전체 순회가 아닌 인덱스 기반으로 변경, 메시마다 다른 Material을 바인딩
- **MODEL enum** — NONANIM/ANIM 구분으로 aiProcess 플래그 분기
- **CForkLift** 신규 Client 클래스 — CModel 기반 게임 오브젝트, 랜덤 배치

| 구분 | 변경 내용 |
|------|-----------|
| 신규 Engine 클래스 | CMaterial |
| 신규 Client 클래스 | CForkLift |
| 수정된 Engine 클래스 | CModel (재설계), CMesh (PreTransform + MaterialIndex) |
| 신규 enum | MODEL (NONANIM/ANIM) |
| 수정된 셰이더 | Shader_VtxMesh.hlsl (g_DiffuseTexture 사용) |
| 핵심 개념 | FBX Material 추출, PreTransformMatrix, 메시-머터리얼 인덱스 매핑 |

> **수업 원본과의 차이:** 수업 코드는 DirectInput8 + 단일 CTransform 구조. 내 프로젝트는 Raw Input + CTransform_3D/2D 분리 (이전 일차에서 이미 적용 완료). 11일차 변경 사항은 구조적 차이 없이 동일하게 적용 가능.

## 1. 이론: FBX Material과 PreTransformMatrix

### 1.1 FBX 파일 내부의 Material 구조

FBX 파일은 메시 데이터뿐 아니라 **재질(Material)** 정보도 함께 저장한다. 하나의 모델에 여러 메시가 있고, 각 메시는 서로 다른 재질을 참조할 수 있다.

```
FBX 파일
├── Mesh 0 (몸통)    → MaterialIndex = 0
├── Mesh 1 (바퀴)    → MaterialIndex = 1
├── Mesh 2 (창문)    → MaterialIndex = 0  (같은 재질 공유 가능)
│
├── Material 0 (몸통/창문 재질)
│   ├── Diffuse Texture: body.png
│   ├── Normal Texture: body_n.png
│   └── Specular Texture: body_s.png
│
└── Material 1 (바퀴 재질)
    └── Diffuse Texture: wheel.png
```

Assimp에서는 이 구조를 다음과 같이 표현한다:

```
aiScene
├── mMeshes[]          — 메시 배열
│   └── aiMesh
│       ├── mVertices, mNormals, mTextureCoords ...
│       └── mMaterialIndex  ← ★ 이 메시가 참조하는 재질 인덱스
│
├── mMaterials[]       — 재질 배열
│   └── aiMaterial
│       ├── GetTextureCount(aiTextureType_DIFFUSE)
│       └── GetTexture(aiTextureType_DIFFUSE, 0, &path)  ← 텍스처 파일 경로
│
└── mNumMeshes, mNumMaterials
```

**핵심 포인트:**
- 메시와 재질은 **인덱스로 연결**된다 (`aiMesh::mMaterialIndex`)
- 하나의 재질에 여러 **텍스처 타입**이 있을 수 있다 (Diffuse, Normal, Specular 등 — `aiTextureType` enum)
- 하나의 타입에도 **여러 장의 텍스처**가 있을 수 있다 (레이어)

### 1.2 메시별 렌더링이 필요한 이유

10일차에서는 `CModel::Render()`가 모든 메시를 한 번에 순회하며 그렸다:

```
Bind_ShaderResources()  ← 셰이더 변수 1회 세팅
Shader::Begin()         ← Pass 1회 적용
CModel::Render()        ← 전체 메시 순회 (내부에서 각 Mesh의 Bind_Resources + DrawIndexed)
```

하지만 **메시마다 다른 텍스처를 사용**해야 하므로, 메시별로 Material을 바인딩한 뒤 개별 렌더해야 한다:

```
Bind_ShaderResources()           ← WVP, 라이트 등 공통 변수 세팅
for (각 메시 i) {
    Bind_Material(i, DIFFUSE)    ← ★ 이 메시의 Diffuse 텍스처 바인딩
    Shader::Begin()              ← Pass 적용
    CModel::Render(i)            ← 이 메시만 DrawIndexed
}
```

### 1.3 PreTransformMatrix — 왜 정점에 직접 적용하는가?

FBX 모델은 제작 도구(3ds Max, Maya 등)마다 좌표계, 스케일, 기본 방향이 다르다. 예를 들어:

| 모델 | 문제 | PreTransformMatrix |
|------|------|--------------------|
| ForkLift.fbx | 100배 크기, 뒷면이 정면 | `Scale(0.01) * RotY(180°)` |
| Fiona.fbx | 뒷면이 정면 | `RotY(180°)` |

이 변환을 런타임 Transform에서 하면 **매 프레임 GPU에서 행렬 곱셈**이 발생한다. 반면 로딩 시점에 CPU에서 정점 데이터에 직접 Bake하면:

- 런타임 오버헤드 없음
- WorldMatrix는 순수하게 게임 로직의 위치/회전/스케일만 담당
- **단, NONANIM(비애니메이션) 모델에만 적용 가능** — 애니메이션 모델은 본(Bone)이 정점을 변환하므로 사전 Bake 불가

```
Position 변환:  XMVector3TransformCoord(position, PreTransformMatrix)    ← 위치 (이동 포함)
Normal 변환:    XMVector3TransformNormal(normal, PreTransformMatrix)     ← 방향만 (이동 무시)
Tangent 변환:   XMVector3TransformNormal(tangent, PreTransformMatrix)    ← 방향만
Binormal 변환:  XMVector3TransformNormal(binormal, PreTransformMatrix)   ← 방향만
```

**핵심 포인트:**
- `TransformCoord`: w=1로 취급, **이동(Translation) 포함**
- `TransformNormal`: w=0으로 취급, **방향 벡터용** (이동 무시, 회전/스케일만 적용)

### 1.4 MODEL enum — NONANIM vs ANIM

```
MODEL::NONANIM  → aiProcess_PreTransformVertices 추가 (노드 계층 구조 평탄화)
MODEL::ANIM     → 추가 안 함 (본 계층 구조 유지, 향후 스키닝에 필요)
```

`aiProcess_PreTransformVertices`는 Assimp가 노드의 로컬 트랜스폼을 정점에 미리 적용하여 **노드 계층을 제거**하는 플래그다. 비애니메이션 모델에서는 계층 구조가 필요 없으므로 이 플래그로 단순화한다.

## 3. CMaterial — FBX 텍스처 추출 클래스

### 3.1 클래스 설계

```cpp
// === Material.h ===
#pragma once

#include "Base.h"

NS_BEGIN(Engine)

class CMaterial final : public CBase
{
private:
    CMaterial(ID3D11Device* pDevice, ID3D11DeviceContext* pContext);
    virtual ~CMaterial() = default;

public:
    // (1) aiMaterial에서 모든 타입의 텍스처를 로딩
    HRESULT Initialize(aiMaterial* pAIMaterial, const _char* pModelFilePath);
    // (2) 특정 타입/인덱스의 SRV를 셰이더에 바인딩
    HRESULT Bind_ShaderResource(class CShader* pShader, const _char* pConstantName, aiTextureType eType, _uint iIndex);

private:
    ID3D11Device* m_pDevice = { nullptr };
    ID3D11DeviceContext* m_pContext = { nullptr };
    // ★ 2차원 배열: [텍스처 타입][텍스처 인덱스]
    vector<ID3D11ShaderResourceView*> m_Materials[AI_TEXTURE_TYPE_MAX];

public:
    static CMaterial* Create(ID3D11Device* pDevice, ID3D11DeviceContext* pContext, aiMaterial* pAIMaterial, const _char* pModelFilePath);
    virtual void Free() override;
};

NS_END
```

**핵심 포인트:**
- CBase 상속 (CComponent 아님) — Prototype/Clone 불필요, CModel 내부에서만 사용
- `m_Materials[AI_TEXTURE_TYPE_MAX]`: aiTextureType별로 SRV 벡터를 보유. 예) `m_Materials[aiTextureType_DIFFUSE]`에 Diffuse 텍스처 SRV들이 들어감

### 3.2 Initialize — 텍스처 경로 조합 + SRV 로딩

```cpp
// === Material.cpp — Initialize() ===
HRESULT CMaterial::Initialize(aiMaterial* pAIMaterial, const _char* pModelFilePath)
{
    _char szDrive[MAX_PATH] = {};
    _char szDir[MAX_PATH] = {};

    // (1) 모델 파일 경로에서 드라이브 + 디렉토리 추출
    _splitpath_s(pModelFilePath, szDrive, MAX_PATH, szDir, MAX_PATH, nullptr, 0, nullptr, 0);

    // (2) 모든 텍스처 타입을 순회 (Diffuse, Normal, Specular 등)
    for (size_t i = 0; i < AI_TEXTURE_TYPE_MAX; i++)
    {
        _uint iNumTextures = pAIMaterial->GetTextureCount(static_cast<aiTextureType>(i));

        for (size_t j = 0; j < iNumTextures; j++)
        {
            aiString strTexturePath = {};
            _char szFileName[MAX_PATH] = {};
            _char szExt[MAX_PATH] = {};

            // (3) aiMaterial에서 텍스처 상대 경로 추출
            if (FAILED(pAIMaterial->GetTexture(static_cast<aiTextureType>(i), j, &strTexturePath)))
                return E_FAIL;

            // (4) 텍스처 파일명 + 확장자 추출
            _splitpath_s(strTexturePath.data, nullptr, 0, nullptr, 0, szFileName, MAX_PATH, szExt, MAX_PATH);

            // (5) 모델 폴더 기준 전체 경로 조합: Drive + Dir + FileName + Ext
            _char szFullPath[MAX_PATH] = {};
            strcpy_s(szFullPath, szDrive);
            strcat_s(szFullPath, szDir);
            strcat_s(szFullPath, szFileName);
            strcat_s(szFullPath, szExt);

            // (6) char → wchar_t 변환 (DirectXTK API가 wchar_t 사용)
            _tchar szFinalPath[MAX_PATH] = {};
            MultiByteToWideChar(CP_ACP, 0, szFullPath, strlen(szFullPath), szFinalPath, MAX_PATH);

            // (7) 확장자별 로딩 분기
            HRESULT hr = {};
            ID3D11ShaderResourceView* pSRV = { nullptr };

            if (false == strcmp(szExt, ".dds"))
                hr = CreateDDSTextureFromFile(m_pDevice, szFinalPath, nullptr, &pSRV);
            else if (false == strcmp(szExt, ".tga"))
                hr = E_FAIL;  // TGA 미지원
            else
                hr = CreateWICTextureFromFile(m_pDevice, szFinalPath, nullptr, &pSRV);

            if (FAILED(hr))
                return E_FAIL;

            // (8) 해당 타입의 SRV 벡터에 추가
            m_Materials[i].push_back(pSRV);
        }
    }

    return S_OK;
}
```

**핵심 포인트:**
- 텍스처 경로가 **상대 경로**로 저장되어 있을 수 있으므로, 모델 파일의 디렉토리를 기준으로 재조합
- `_splitpath_s`를 2번 사용: (1) 모델 경로에서 드라이브+디렉토리, (2) 텍스처 경로에서 파일명+확장자
- Assimp는 char(멀티바이트)를 사용하지만, DirectXTK는 wchar_t를 사용하므로 `MultiByteToWideChar` 변환 필요

### 3.3 텍스처 경로 조합 흐름 (시각화)

```
모델 파일 경로:   "../Bin/Resources/Models/ForkLift/ForkLift.fbx"
                   \_________drive+dir__________/ \__무시__/

aiMaterial 텍스처: "textures/body_diffuse.png"
                    \__무시__/ \__fileName__/ \.ext/

최종 경로: "../Bin/Resources/Models/ForkLift/" + "body_diffuse" + ".png"
            \_________drive+dir______________/   \__fileName__/  \ext/
```

### 3.4 Bind_ShaderResource

```cpp
// === Material.cpp — Bind_ShaderResource() ===
HRESULT CMaterial::Bind_ShaderResource(CShader* pShader, const _char* pConstantName, aiTextureType eType, _uint iIndex)
{
    // 범위 검사
    if (eType >= AI_TEXTURE_TYPE_MAX ||
        iIndex >= m_Materials[eType].size())
        return E_FAIL;

    // CShader::Bind_SRV로 위임
    return pShader->Bind_SRV(pConstantName, m_Materials[eType][iIndex]);
}
```

### 3.5 Free

```cpp
// === Material.cpp — Free() ===
void CMaterial::Free()
{
    __super::Free();

    // 모든 타입의 모든 SRV 해제
    for (auto& SRVs : m_Materials)
    {
        for (auto& pSRV : SRVs)
            Safe_Release(pSRV);
        SRVs.clear();
    }

    Safe_Release(m_pDevice);
    Safe_Release(m_pContext);
}
```

### 4.2 Initialize_Prototype — 정점 변환 적용

**Before (10일차):**
```cpp
// 단순 memcpy만
memcpy(&pVertices[i].vPosition, &pAIMesh->mVertices[i], sizeof(_float3));
memcpy(&pVertices[i].vNormal, &pAIMesh->mNormals[i], sizeof(_float3));
memcpy(&pVertices[i].vTexcoord, &pAIMesh->mTextureCoords[0][i], sizeof(_float2));
memcpy(&pVertices[i].vTangent, &pAIMesh->mTangents[i], sizeof(_float3));
memcpy(&pVertices[i].vBinormal, &pAIMesh->mBitangents[i], sizeof(_float3));
```

**After (11일차):**
```cpp
// === Mesh.cpp — Initialize_Prototype() ===
HRESULT CMesh::Initialize_Prototype(const aiMesh* pAIMesh, _fmatrix PreTransformMatrix)
{
    // ★ NEW: Material 인덱스 저장
    m_iMaterialIndex = pAIMesh->mMaterialIndex;

    // ... (버퍼 설정은 동일) ...

    for (size_t i = 0; i < m_iNumVertices; i++)
    {
        // Position: memcpy 후 TransformCoord (이동 포함 변환)
        memcpy(&pVertices[i].vPosition, &pAIMesh->mVertices[i], sizeof(_float3));
        XMStoreFloat3(&pVertices[i].vPosition,
            XMVector3TransformCoord(XMLoadFloat3(&pVertices[i].vPosition), PreTransformMatrix));  // ★ NEW

        // Normal: memcpy 후 TransformNormal (방향만 변환)
        memcpy(&pVertices[i].vNormal, &pAIMesh->mNormals[i], sizeof(_float3));
        XMStoreFloat3(&pVertices[i].vNormal,
            XMVector3TransformNormal(XMLoadFloat3(&pVertices[i].vNormal), PreTransformMatrix));   // ★ NEW

        // Texcoord: 변환 없음 (UV는 2D 좌표)
        memcpy(&pVertices[i].vTexcoord, &pAIMesh->mTextureCoords[0][i], sizeof(_float2));

        // Tangent: TransformNormal
        memcpy(&pVertices[i].vTangent, &pAIMesh->mTangents[i], sizeof(_float3));
        XMStoreFloat3(&pVertices[i].vTangent,
            XMVector3TransformNormal(XMLoadFloat3(&pVertices[i].vTangent), PreTransformMatrix));  // ★ NEW

        // Binormal: TransformNormal
        memcpy(&pVertices[i].vBinormal, &pAIMesh->mBitangents[i], sizeof(_float3));
        XMStoreFloat3(&pVertices[i].vBinormal,
            XMVector3TransformNormal(XMLoadFloat3(&pVertices[i].vBinormal), PreTransformMatrix)); // ★ NEW
    }

    // ... (CreateBuffer는 동일) ...
}
```

**핵심 포인트:**
- `memcpy` → `XMVector3Transform*` 순서: 먼저 Assimp 데이터를 복사한 뒤 변환 적용
- Texcoord(UV)에는 PreTransformMatrix를 적용하지 않음 — UV는 텍스처 좌표이므로 3D 변환과 무관

### 5.5 Bind_Material — 메시 → Material 인덱스 매핑

```cpp
// === Model.cpp — Bind_Material() ===
HRESULT CModel::Bind_Material(CShader* pShader, const _char* pConstantName, _uint iMeshIndex, aiTextureType eType, _uint iIndex)
{
    if (iMeshIndex >= m_iNumMeshes)
        return E_FAIL;

    // (1) 메시가 참조하는 Material 인덱스 획득
    _uint iMaterialIndex = m_Meshes[iMeshIndex]->Get_MaterialIndex();
    if (iMaterialIndex >= m_iNumMaterials)
        return E_FAIL;

    // (2) 해당 Material의 SRV를 셰이더에 바인딩
    return m_Materials[iMaterialIndex]->Bind_ShaderResource(pShader, pConstantName, eType, iIndex);
}
```

**데이터 흐름:**
```
GameObject::Render()
    ↓ Bind_Material(shader, "g_DiffuseTexture", meshIdx=1, DIFFUSE, 0)
CModel::Bind_Material()
    ↓ m_Meshes[1]->Get_MaterialIndex()  →  materialIdx = 2
    ↓ m_Materials[2]->Bind_ShaderResource(shader, "g_DiffuseTexture", DIFFUSE, 0)
CMaterial::Bind_ShaderResource()
    ↓ m_Materials[DIFFUSE][0]  →  SRV 포인터
    ↓ pShader->Bind_SRV("g_DiffuseTexture", pSRV)
```

### 7.1 Render — 메시별 Material 바인딩 + 개별 렌더링

```cpp
// === ForkLift.cpp — Render() ===
HRESULT CForkLift::Render()
{
    // (1) 공통 셰이더 변수 세팅 (WVP, 라이트 등)
    if (FAILED(Bind_ShaderResources()))
        return E_FAIL;

    // (2) 메시 수 조회
    size_t iNumMeshes = m_pModelCom->Get_NumMeshes();

    // (3) ★ 메시별 루프: Material 바인딩 → Pass 적용 → 렌더
    for (size_t i = 0; i < iNumMeshes; i++)
    {
        // 이 메시의 Diffuse 텍스처를 "g_DiffuseTexture"에 바인딩
        if (FAILED(m_pModelCom->Bind_Material(m_pShaderCom, "g_DiffuseTexture", i, aiTextureType_DIFFUSE, 0)))
            return E_FAIL;

        // Pass 적용 (Material이 바뀌었으므로 매 메시마다 Begin 필요)
        if (FAILED(m_pShaderCom->Begin(0)))
            return E_FAIL;

        // 이 메시만 DrawIndexed
        if (FAILED(m_pModelCom->Render(i)))
            return E_FAIL;
    }

    return S_OK;
}
```

**핵심 포인트:**
- `Bind_ShaderResources()`는 루프 **밖에서 1회** — WVP, 라이트는 모든 메시에 공통
- `Bind_Material()` + `Begin()` + `Render()`는 루프 **안에서 매 메시마다** — 텍스처가 다르면 Pass를 다시 적용해야 함
- `Begin(0)`을 매 메시마다 호출하는 이유: `Apply()`가 변경된 셰이더 변수(텍스처)를 GPU에 반영하기 때문

### 7.2 10일차 Render()와의 비교

| 항목 | 10일차 | 11일차 |
|------|--------|--------|
| Bind_ShaderResources | 루프 밖 1회 | 루프 밖 1회 (동일) |
| 텍스처 바인딩 | 없음 (셰이더에서 1.f) | Bind_Material로 메시별 |
| Shader::Begin | 루프 밖 1회 | **루프 안 매 메시마다** |
| Model::Render | 전체 순회 1회 호출 | 인덱스 기반 개별 호출 |

## 10. Shader_VtxMesh.hlsl — g_DiffuseTexture 사용

**Before (10일차):**
```hlsl
// 텍스처 없이 고정값 사용
vector vMtrlDiffuse = vector(1.f, 1.f, 1.f, 1.f);
```

**After (11일차):**
```hlsl
// === Shader_VtxMesh.hlsl — PS_MAIN() ===

// ★ 재질 텍스처 변수
texture2D g_DiffuseTexture;

PS_OUT PS_MAIN(PS_IN In)
{
    PS_OUT Out;

    // ★ Material에서 바인딩된 Diffuse 텍스처 샘플링
    vector vMtrlDiffuse = g_DiffuseTexture.Sample(DefaultSampler, In.vTexcoord);
    if (vMtrlDiffuse.a < 0.1f)
        discard;

    // Phong Lighting (이전과 동일)
    vector vShade = max(dot(normalize(g_vLightDir) * -1.f, In.vNormal), 0.f)
                  + (g_vLightAmbient * g_vMtrlAmbient);

    vector vReflect = reflect(normalize(g_vLightDir), normalize(In.vNormal));
    vector vLook = In.vWorldPos - g_vCamPosition;

    float fSpecular = pow(max(dot(normalize(vLook) * -1.f, normalize(vReflect)), 0.f), 50.f);

    Out.vColor = g_vLightDiffuse * vMtrlDiffuse * saturate(vShade) +
        (g_vLightSpecular * g_vMtrlSpecular) * fSpecular;

    return Out;
}
```

**핵심 포인트:**
- 셰이더 변수명이 `g_Texture`에서 `g_DiffuseTexture`로 변경 — 향후 Normal/Specular 텍스처와 구분
- `Shader_VtxNorTex.hlsl`도 동일하게 `g_DiffuseTexture` 사용 (Terrain용)
