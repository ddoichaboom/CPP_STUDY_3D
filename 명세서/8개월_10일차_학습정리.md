9일차에서 CModel의 기초 골격(aiScene 로딩)만 구현했다면, 10일차에서는 **aiScene에서 실제 메시 데이터를 추출**하여 GPU 렌더링 가능한 상태까지 완성한다.

## 1. 이론: 3D 모델 파이프라인 — aiScene에서 GPU까지

### 1.1 9일차까지의 한계

9일차에서는 Assimp으로 FBX 파일을 로드하여 `aiScene*`을 얻었지만, 실제로 **메시 데이터를 추출하지 않았다**. aiScene은 CPU 메모리에 있는 Assimp 내부 자료구조일 뿐, D3D11이 렌더링하려면 **GPU의 Vertex/Index Buffer**가 필요하다.

### 1.2 10일차의 데이터 흐름

```
FBX 파일 (디스크)
    ↓ Assimp::Importer::ReadFile()
aiScene (CPU — Assimp 내부 구조)
    ├── mMeshes[0]: aiMesh (정점/면/UV/노멀/탄젠트 데이터)
    ├── mMeshes[1]: aiMesh
    └── ...mMeshes[N-1]
    ↓ CModel::Ready_Meshes()
    ↓   각 aiMesh마다 CMesh::Create()
CMesh (CVIBuffer 상속)
    ├── ID3D11Buffer* m_pVB (GPU Vertex Buffer)
    └── ID3D11Buffer* m_pIB (GPU Index Buffer)
    ↓ CModel::Render() → CMesh::Bind_Resources() + CMesh::Render()
GPU 파이프라인 (IA → VS → RS → PS → OM)
```

### 1.3 핵심 개념: 하나의 모델 ≠ 하나의 메시

FBX 모델 파일 하나에 **여러 개의 메시**가 포함될 수 있다. 예를 들어:
- 캐릭터 모델: 몸통, 머리, 팔, 다리가 각각 별도 메시
- 차량 모델(ForkLift): 차체, 바퀴, 포크가 별도 메시

따라서 `CModel`은 `vector<CMesh*>`로 여러 메시를 관리하고, `Render()` 시 모든 메시를 순회하며 그린다.

### 1.4 후처리 플래그 변경 (9일차 vs 10일차)

```cpp
// 9일차
_uint iFlag = aiProcess_ConvertToLeftHanded | aiProcessPreset_TargetRealtime_Fast;

// 10일차 — 두 가지 플래그 추가
_uint iFlag = aiProcess_GlobalScale |
              aiProcess_PreTransformVertices |
              aiProcess_ConvertToLeftHanded |
              aiProcessPreset_TargetRealtime_Fast;
```

| 플래그 | 설명 |
|--------|------|
| `aiProcess_GlobalScale` | 모델의 스케일을 정규화. FBX 파일마다 단위가 다를 수 있음 (cm, m, inch 등) |
| `aiProcess_PreTransformVertices` | 노드 계층 구조를 평탄화하여 모든 변환을 정점에 미리 적용. **애니메이션이 없는 static 모델**에 적합 |
| `aiProcess_ConvertToLeftHanded` | DirectX 왼손 좌표계로 변환 (9일차와 동일) |
| `aiProcessPreset_TargetRealtime_Fast` | 실시간 렌더링 전처리 묶음 (9일차와 동일) |

**`aiProcess_PreTransformVertices`의 의미:**
- FBX의 노드 트리에서 부모-자식 변환 행렬을 정점에 미리 곱함
- 결과: 모든 정점이 **모델 로컬 공간** 기준으로 통일
- 장점: 노드 순회 불필요, 렌더링 단순화
- 단점: **애니메이션 정보가 소실됨** (뼈대 계층 구조가 사라지므로)
- 따라서 static 모델(건물, 소품, 차량 등)에만 사용하고, 캐릭터 애니메이션 모델에는 사용하지 않음

## 2. VTXMESH 정점 포맷 — Engine_Struct.h

### 2.1 기존 정점 포맷과 비교

| 포맷 | 구성 | 크기 | 용도 |
|------|------|------|------|
| `VTXTEX` | Position + Texcoord | 20 bytes | UI 사각형 (CVIBuffer_Rect) |
| `VTXNORTEX` | Position + Normal + Texcoord | 32 bytes | Terrain (노멀 라이팅) |
| `VTXMESH` | Position + Normal + Texcoord + Tangent + Binormal | 56 bytes | 3D 모델 (FBX) |

### 2.2 Tangent와 Binormal이란?

노멀 매핑(Normal Mapping)을 위해 **TBN 행렬**(Tangent-Binormal-Normal)이 필요하다:

```
Normal (법선)    — 표면에 수직인 벡터 (이미 8일차에서 학습)
Tangent (접선)   — 텍스처 U 방향으로의 접선 벡터
Binormal (종법선) — Normal × Tangent로 계산되는 벡터 (텍스처 V 방향)
```

이 세 벡터가 각 정점에서 **접선 공간(Tangent Space)**을 형성한다. 노멀맵의 RGB 값을 이 공간에서 월드 공간으로 변환할 때 사용한다.

> 현재(10일차)에서는 Tangent/Binormal을 **저장만** 하고, 셰이더에서 실제 노멀 매핑 계산은 하지 않는다. 향후 수업에서 구현할 예비 작업.

### 2.3 VTXMESH 구조체 코드

```cpp
// === Engine_Struct.h — VTXNORTEX 아래에 추가 ===

typedef struct tagVertexMesh
{
    XMFLOAT3        vPosition;      // 12 bytes (offset 0)
    XMFLOAT3        vNormal;        // 12 bytes (offset 12)
    XMFLOAT2        vTexcoord;      //  8 bytes (offset 24)
    XMFLOAT3        vTangent;       // 12 bytes (offset 32)
    XMFLOAT3        vBinormal;      // 12 bytes (offset 44)
                                    // Total: 56 bytes

    static const unsigned int       iNumElements = { 5 };

    static constexpr D3D11_INPUT_ELEMENT_DESC   Elements[] = {
        {"POSITION", 0, DXGI_FORMAT_R32G32B32_FLOAT, 0, 0,  D3D11_INPUT_PER_VERTEX_DATA, 0},
        {"NORMAL",   0, DXGI_FORMAT_R32G32B32_FLOAT, 0, 12, D3D11_INPUT_PER_VERTEX_DATA, 0},
        {"TEXCOORD", 0, DXGI_FORMAT_R32G32_FLOAT,    0, 24, D3D11_INPUT_PER_VERTEX_DATA, 0},
        {"TANGENT",  0, DXGI_FORMAT_R32G32B32_FLOAT, 0, 32, D3D11_INPUT_PER_VERTEX_DATA, 0},
        {"BINORMAL", 0, DXGI_FORMAT_R32G32B32_FLOAT, 0, 44, D3D11_INPUT_PER_VERTEX_DATA, 0},
    };
}VTXMESH;
```

**핵심 포인트:**
- Texcoord는 `XMFLOAT2` (8 bytes) — Assimp의 `mTextureCoords`는 XMFLOAT3이지만 **UV만 사용** (W 무시)
- Elements의 offset 값이 각 필드의 바이트 시작 위치와 정확히 일치해야 함
- Tangent/Binormal은 Assimp이 `aiProcessPreset_TargetRealtime_Fast` 내부의 `aiProcess_CalcTangentSpace`로 자동 생성

## 3. CMesh 클래스 — aiMesh를 GPU 버퍼로 변환

### 3.1 설계 원칙

- `CVIBuffer` 상속 → ID3D11Buffer(VB/IB) 관리, Bind_Resources/Render 재사용
- aiMesh 데이터를 VTXMESH 형식으로 변환하여 GPU 버퍼 생성
- **Clone()은 nullptr 반환** — 메시는 CModel 단위로 공유됨 (개별 클론 불필요)

### 3.2 Mesh.h

```cpp
#pragma once

#include "VIBuffer.h"

NS_BEGIN(Engine)

class CMesh final : public CVIBuffer
{
private:
    CMesh(ID3D11Device* pDevice, ID3D11DeviceContext* pContext);
    CMesh(const CMesh& Prototype);
    virtual ~CMesh() = default;

public:
    virtual HRESULT Initialize_Prototype(const aiMesh* pAIMesh);
    virtual HRESULT Initialize(void* pArg);

public:
    static CMesh* Create(ID3D11Device* pDevice, ID3D11DeviceContext* pContext, const aiMesh* pAIMesh);
    virtual CComponent* Clone(void* pArg) override;
    virtual void Free() override;
};

NS_END
```

**특이점:**
- `Initialize_Prototype`의 인자가 `const aiMesh*` — 기존 컴포넌트와 다른 시그니처
- `ENGINE_DLL` 미사용 — CModel 내부에서만 사용, Client에서 직접 접근하지 않음
- `Clone()` → `nullptr` — 메시는 Model 복사 생성자에서 `Safe_AddRef`로 공유

### 3.3 Mesh.cpp

```cpp
#include "Mesh.h"

CMesh::CMesh(ID3D11Device* pDevice, ID3D11DeviceContext* pContext)
    : CVIBuffer{ pDevice, pContext }
{
}

CMesh::CMesh(const CMesh& Prototype)
    : CVIBuffer{ Prototype }
{
}

HRESULT CMesh::Initialize_Prototype(const aiMesh* pAIMesh)
{
    // === CVIBuffer 멤버 변수 설정 ===
    m_iNumVertexBuffers = 1;
    m_iNumVertices = pAIMesh->mNumVertices;
    m_iVertexStride = sizeof(VTXMESH);

    m_iNumIndices = pAIMesh->mNumFaces * 3;     // 각 face는 삼각형 (3 인덱스)
    m_iIndexStride = 4;                          // 32-bit index
    m_eIndexFormat = DXGI_FORMAT_R32_UINT;

    m_ePrimitiveType = D3D11_PRIMITIVE_TOPOLOGY_TRIANGLELIST;

    // === Vertex Buffer 생성 ===
    D3D11_BUFFER_DESC           VertexBufferDesc{};
    VertexBufferDesc.ByteWidth = m_iVertexStride * m_iNumVertices;
    VertexBufferDesc.Usage = D3D11_USAGE_DEFAULT;
    VertexBufferDesc.BindFlags = D3D11_BIND_VERTEX_BUFFER;
    VertexBufferDesc.CPUAccessFlags = 0;
    VertexBufferDesc.MiscFlags = 0;
    VertexBufferDesc.StructureByteStride = m_iVertexStride;

    VTXMESH* pVertices = new VTXMESH[m_iNumVertices];
    ZeroMemory(pVertices, sizeof(VTXMESH) * m_iNumVertices);

    for (size_t i = 0; i < m_iNumVertices; i++)
    {
        memcpy(&pVertices[i].vPosition, &pAIMesh->mVertices[i], sizeof(_float3));
        memcpy(&pVertices[i].vNormal, &pAIMesh->mNormals[i], sizeof(_float3));
        memcpy(&pVertices[i].vTexcoord, &pAIMesh->mTextureCoords[0][i], sizeof(_float2));
        memcpy(&pVertices[i].vTangent, &pAIMesh->mTangents[i], sizeof(_float3));
        memcpy(&pVertices[i].vBinormal, &pAIMesh->mBitangents[i], sizeof(_float3));
    }

    D3D11_SUBRESOURCE_DATA      VertexInitialData{};
    VertexInitialData.pSysMem = pVertices;

    if (FAILED(m_pDevice->CreateBuffer(&VertexBufferDesc, &VertexInitialData, &m_pVB)))
        return E_FAIL;

    Safe_Delete_Array(pVertices);

    // === Index Buffer 생성 ===
    D3D11_BUFFER_DESC           IndexBufferDesc{};
    IndexBufferDesc.ByteWidth = m_iIndexStride * m_iNumIndices;
    IndexBufferDesc.Usage = D3D11_USAGE_DEFAULT;
    IndexBufferDesc.BindFlags = D3D11_BIND_INDEX_BUFFER;
    IndexBufferDesc.CPUAccessFlags = 0;
    IndexBufferDesc.MiscFlags = 0;
    IndexBufferDesc.StructureByteStride = m_iIndexStride;

    _uint* pIndices = new _uint[m_iNumIndices];
    ZeroMemory(pIndices, sizeof(_uint) * m_iNumIndices);

    _uint       iNumIndices = {};

    for (size_t i = 0; i < pAIMesh->mNumFaces; i++)
    {
        pIndices[iNumIndices++] = pAIMesh->mFaces[i].mIndices[0];
        pIndices[iNumIndices++] = pAIMesh->mFaces[i].mIndices[1];
        pIndices[iNumIndices++] = pAIMesh->mFaces[i].mIndices[2];
    }

    D3D11_SUBRESOURCE_DATA      IndexInitialData{};
    IndexInitialData.pSysMem = pIndices;

    if (FAILED(m_pDevice->CreateBuffer(&IndexBufferDesc, &IndexInitialData, &m_pIB)))
        return E_FAIL;

    Safe_Delete_Array(pIndices);

    return S_OK;
}

HRESULT CMesh::Initialize(void* pArg)
{
    return S_OK;
}

CMesh* CMesh::Create(ID3D11Device* pDevice, ID3D11DeviceContext* pContext, const aiMesh* pAIMesh)
{
    CMesh* pInstance = new CMesh(pDevice, pContext);

    if (FAILED(pInstance->Initialize_Prototype(pAIMesh)))
    {
        MSG_BOX("Failed to Created : CMesh");
        Safe_Release(pInstance);
    }

    return pInstance;
}

CComponent* CMesh::Clone(void* pArg)
{
    // 메시는 개별 클론 불필요 — CModel 복사 생성자에서 공유
    return nullptr;
}

void CMesh::Free()
{
    __super::Free();
    // CVIBuffer::Free()가 m_pVB, m_pIB를 해제
}
```

### 3.4 aiMesh → VTXMESH 변환 상세

```
aiMesh 멤버                          →  VTXMESH 필드
─────────────────────────────────────────────────────
pAIMesh->mVertices[i]      (aiVector3D)  →  vPosition  (XMFLOAT3, 12 bytes 복사)
pAIMesh->mNormals[i]       (aiVector3D)  →  vNormal    (XMFLOAT3, 12 bytes 복사)
pAIMesh->mTextureCoords[0][i] (aiVector3D) → vTexcoord (XMFLOAT2, 8 bytes만 복사!)
pAIMesh->mTangents[i]      (aiVector3D)  →  vTangent   (XMFLOAT3, 12 bytes 복사)
pAIMesh->mBitangents[i]    (aiVector3D)  →  vBinormal  (XMFLOAT3, 12 bytes 복사)
```

**주의사항:**
- `mTextureCoords[0]`의 `[0]`은 **첫 번째 UV 채널** — FBX 모델은 여러 UV 채널을 가질 수 있음
- `aiVector3D`는 `{x, y, z}` 12 bytes이지만, `vTexcoord`는 `XMFLOAT2` 8 bytes → **sizeof(_float2)로 복사**하여 z 값을 자연스럽게 무시
- `mBitangents` → `vBinormal`: Assimp은 "Bitangent"이라 부르고, 엔진에서는 "Binormal"이라 부름 (동일한 개념)

### 3.5 인덱스 추출

```
aiMesh->mFaces[i]
    ├── mNumIndices = 3 (삼각형 — aiProcess_Triangulate가 보장)
    └── mIndices[0], mIndices[1], mIndices[2]

mFaces 수 × 3 = 전체 인덱스 수
```

삼각형화(Triangulate)는 `aiProcessPreset_TargetRealtime_Fast`에 포함되어 있으므로, 각 face는 반드시 3개의 인덱스를 가진다.

## 4. CModel 확장 — Ready_Meshes, Render, 복사 생성자

### 4.1 변경 요약 (9일차 → 10일차)

| 항목 | 9일차 | 10일차 |
|------|-------|--------|
| 멤버 변수 | `m_pAIScene`, `m_Importer`만 | + `m_iNumMeshes`, `m_Meshes` |
| Initialize_Prototype | ReadFile만 | + `Ready_Meshes()` 호출 |
| 복사 생성자 | 빈 상태 | 메시 벡터 복사 + Safe_AddRef |
| Render() | 없음 | 모든 메시 순회하며 Bind+Render |
| Free() | 빈 상태 | 메시 해제 + `m_Importer.FreeScene()` |
| 후처리 플래그 | 2개 | + `aiProcess_GlobalScale` + `aiProcess_PreTransformVertices` |

### 4.2 Model.h (10일차)

### 4.3 Model.cpp (10일차)

```cpp
#include "Model.h"
#include "Mesh.h"

CModel::CModel(ID3D11Device* pDevice, ID3D11DeviceContext* pContext)
    : CComponent{ pDevice, pContext }
{
}

// ★ 복사 생성자 — 메시 벡터 공유
CModel::CModel(const CModel& Prototype)
    : CComponent{ Prototype }
    , m_iNumMeshes{ Prototype.m_iNumMeshes }
    , m_Meshes{ Prototype.m_Meshes }
{
    // 공유하는 메시들의 참조 카운트 증가
    for (auto& pMesh : m_Meshes)
        Safe_AddRef(pMesh);
}

HRESULT CModel::Initialize_Prototype(const _char* pModelFilePath)
{
    // ★ 후처리 플래그 — GlobalScale + PreTransformVertices 추가
    _uint iFlag = { aiProcess_GlobalScale |
                    aiProcess_PreTransformVertices |
                    aiProcess_ConvertToLeftHanded |
                    aiProcessPreset_TargetRealtime_Fast };

    m_pAIScene = m_Importer.ReadFile(pModelFilePath, iFlag);
    if (nullptr == m_pAIScene)
        return E_FAIL;

    // ★ NEW: aiScene에서 메시 데이터 추출
    if (FAILED(Ready_Meshes()))
        return E_FAIL;

    return S_OK;
}

HRESULT CModel::Initialize(void* pArg)
{
    return S_OK;
}

// ★ NEW: 모든 메시를 순회하며 렌더링
HRESULT CModel::Render()
{
    for (auto& pMesh : m_Meshes)
    {
        pMesh->Bind_Resources();        // IA 스테이지에 VB/IB 바인딩
        pMesh->Render();                 // DrawIndexed 호출
    }

    return S_OK;
}

// ★ NEW: aiScene의 모든 메시를 CMesh로 변환
HRESULT CModel::Ready_Meshes()
{
    m_iNumMeshes = m_pAIScene->mNumMeshes;

    for (size_t i = 0; i < m_iNumMeshes; i++)
    {
        CMesh* pMesh = CMesh::Create(m_pDevice, m_pContext, m_pAIScene->mMeshes[i]);
        if (nullptr == pMesh)
            return E_FAIL;

        m_Meshes.push_back(pMesh);
    }

    return S_OK;
}

CModel* CModel::Create(ID3D11Device* pDevice, ID3D11DeviceContext* pContext, const _char* pModelFilePath)
{
    CModel* pInstance = new CModel(pDevice, pContext);

    if (FAILED(pInstance->Initialize_Prototype(pModelFilePath)))
    {
        MSG_BOX("Failed to Created : CModel");
        Safe_Release(pInstance);
    }

    return pInstance;
}

CComponent* CModel::Clone(void* pArg)
{
    CModel* pInstance = new CModel(*this);

    if (FAILED(pInstance->Initialize(pArg)))
    {
        MSG_BOX("Failed to Cloned : CModel");
        Safe_Release(pInstance);
    }

    return pInstance;
}

// ★ Free 변경 — 메시 해제 + Importer 정리
void CModel::Free()
{
    __super::Free();

    for (auto& pMesh : m_Meshes)
        Safe_Release(pMesh);

    m_Meshes.clear();

    m_Importer.FreeScene();     // Importer가 소유한 aiScene 명시적 해제
}
```

### 4.4 메시 공유 패턴 분석

```
CModel (Prototype)                      CModel (Clone)
┌─────────────────┐                     ┌─────────────────┐
│ m_Meshes        │                     │ m_Meshes        │
│   [0] ──────────┼───┐            ┌────┼── [0]           │
│   [1] ──────────┼─┐ │            │ ┌──┼── [1]           │
└─────────────────┘ │ │            │ │  └─────────────────┘
                    │ │            │ │
                    │ └──► CMesh A ◄──┘  (RefCnt = 2)
                    └────► CMesh B ◄─┘   (RefCnt = 2)
```

- Prototype가 실제 GPU 버퍼를 소유
- Clone은 같은 CMesh 포인터를 공유 (Safe_AddRef)
- Clone에서 Free() 시 참조 카운트만 감소
- Prototype에서 Free() 시 참조 카운트 0 → GPU 버퍼 실제 해제

### 4.5 `m_Importer.FreeScene()`

9일차에서는 Importer 소멸자가 aiScene을 자동 해제했지만, 10일차에서는 **명시적으로 `FreeScene()`을 호출**한다.

이유: Ready_Meshes()로 메시 데이터를 이미 GPU 버퍼로 복사했으므로, aiScene의 원본 데이터는 더 이상 필요 없다. Free()에서 조기 해제하여 CPU 메모리를 절약한다.

## 6. Shader_VtxMesh.hlsl — VTXMESH용 셰이더

### 6.1 Shader_VtxNorTex.hlsl과의 차이

| 항목 | Shader_VtxNorTex | Shader_VtxMesh |
|------|------------------|----------------|
| VS_IN | Position + Normal + Texcoord | + **Tangent** + **Binormal** |
| VS_OUT | 동일 | 동일 |
| PS_MAIN 텍스처 | `g_DiffuseTexture.Sample()` | **`1.f`** (하드코딩 흰색) |
| PS 라이팅 | Phong | Phong (동일) |

### 6.2 VS_IN 변경

```hlsl
struct VS_IN
{
    float3 vPosition : POSITION;
    float3 vNormal : NORMAL;
    float2 vTexcoord : TEXCOORD0;

    // ★ VTXMESH 전용
    float3 vTangent : TANGENT;
    float3 vBinormal : BINORMAL;
};
```

Tangent/Binormal은 현재 셰이더 로직에서 사용하지 않지만, **InputLayout과 정점 구조체가 1:1 대응**해야 하므로 VS_IN에 선언이 필요하다. 선언하지 않으면 InputLayout 생성 실패.

### 6.3 PS_MAIN — 텍스처 미사용

```hlsl
PS_OUT PS_MAIN(PS_IN In)
{
    PS_OUT Out;

    // ★ 텍스처 대신 흰색(1.f) 사용
    vector vMtrlDiffuse = /*g_DiffuseTexture.Sample(DefaultSampler, In.vTexcoord)*/ 1.f;
    if (vMtrlDiffuse.a < 0.1f)
        discard;

    // 이하 Phong 라이팅 계산은 Shader_VtxNorTex와 동일
    vector vShade = max(dot(normalize(g_vLightDir) * -1.f, In.vNormal), 0.f)
                    + (g_vLightAmbient * g_vMtrlAmbient);

    vector vReflect = reflect(normalize(g_vLightDir), normalize(In.vNormal));
    vector vLook = In.vWorldPos - g_vCamPosition;

    float fSpecular = pow(max(dot(normalize(vLook) * -1.f, normalize(vReflect)), 0.f), 50.f);

    Out.vColor = g_vLightDiffuse * vMtrlDiffuse * saturate(vShade)
                 + (g_vLightSpecular * g_vMtrlSpecular) * fSpecular;

    return Out;
}
```

**왜 텍스처를 사용하지 않는가?**
현재 단계에서는 CModel이 aiScene에서 **텍스처를 추출하는 기능이 없다**. 메시(정점/인덱스)만 추출한다. 모델 텍스처 로딩은 향후 수업에서 `aiMaterial`을 처리할 때 구현될 예정.

### 6.4 전체 코드

```hlsl
float4x4 g_WorldMatrix, g_ViewMatrix, g_ProjMatrix;

vector g_vCamPosition;

// 빛
vector g_vLightDir;
vector g_vLightDiffuse;
vector g_vLightAmbient;
vector g_vLightSpecular;

// 재질
texture2D g_DiffuseTexture;
vector g_vMtrlAmbient  = vector(0.4f, 0.4f, 0.4f, 1.f);
vector g_vMtrlSpecular = vector(1.f, 1.f, 1.f, 1.f);

sampler DefaultSampler = sampler_state
{
    Filter = MIN_MAG_MIP_LINEAR;
    AddressU = wrap;
    AddressV = wrap;
};

struct VS_IN
{
    float3 vPosition : POSITION;
    float3 vNormal   : NORMAL;
    float2 vTexcoord : TEXCOORD0;
    float3 vTangent  : TANGENT;
    float3 vBinormal : BINORMAL;
};

struct VS_OUT
{
    float4 vPosition : SV_POSITION;
    float4 vNormal   : NORMAL;
    float2 vTexcoord : TEXCOORD0;
    float4 vWorldPos : TEXCOORD1;
};

VS_OUT VS_MAIN(VS_IN In)
{
    VS_OUT Out;

    float4 vPosition = mul(float4(In.vPosition, 1.f), g_WorldMatrix);
    vPosition = mul(vPosition, g_ViewMatrix);
    vPosition = mul(vPosition, g_ProjMatrix);

    Out.vPosition = vPosition;
    Out.vNormal = normalize(mul(float4(In.vNormal, 0.f), g_WorldMatrix));
    Out.vTexcoord = In.vTexcoord;
    Out.vWorldPos = mul(float4(In.vPosition, 1.f), g_WorldMatrix);

    return Out;
}

struct PS_IN
{
    float4 vPosition : SV_POSITION;
    float4 vNormal   : NORMAL;
    float2 vTexcoord : TEXCOORD0;
    float4 vWorldPos : TEXCOORD1;
};

struct PS_OUT
{
    float4 vColor : SV_TARGET0;
};

PS_OUT PS_MAIN(PS_IN In)
{
    PS_OUT Out;

    vector vMtrlDiffuse = /*g_DiffuseTexture.Sample(DefaultSampler, In.vTexcoord)*/ 1.f;
    if (vMtrlDiffuse.a < 0.1f)
        discard;

    vector vShade = max(dot(normalize(g_vLightDir) * -1.f, In.vNormal), 0.f)
                    + (g_vLightAmbient * g_vMtrlAmbient);

    vector vReflect = reflect(normalize(g_vLightDir), normalize(In.vNormal));
    vector vLook = In.vWorldPos - g_vCamPosition;

    float fSpecular = pow(max(dot(normalize(vLook) * -1.f, normalize(vReflect)), 0.f), 50.f);

    Out.vColor = g_vLightDiffuse * vMtrlDiffuse * saturate(vShade)
                 + (g_vLightSpecular * g_vMtrlSpecular) * fSpecular;

    return Out;
}

technique11 DefaultTechnique
{
    pass DefaultPass
    {
        VertexShader = compile vs_5_0 VS_MAIN();
        PixelShader  = compile ps_5_0 PS_MAIN();
    }
}
```

## 10. CModel 렌더링 vs CVIBuffer 렌더링 비교

### 10.1 CVIBuffer 기반 (Terrain)

```cpp
// Terrain::Render()
Bind_ShaderResources();         // WVP + 텍스처 + 라이트 바인딩
m_pShaderCom->Begin(0);         // 셰이더 Pass 적용
m_pVIBufferCom->Bind_Resources();   // IA에 VB/IB 세팅
m_pVIBufferCom->Render();           // DrawIndexed()
```

하나의 VIBuffer = 하나의 DrawCall.

### 10.2 CModel 기반 (Monster)

```cpp
// Monster::Render()
Bind_ShaderResources();         // WVP + 라이트 바인딩 (텍스처 없음)
m_pShaderCom->Begin(0);         // 셰이더 Pass 적용
m_pModelCom->Render();          // ★ 내부에서 메시 수만큼 DrawCall

// CModel::Render() 내부:
for (auto& pMesh : m_Meshes)
{
    pMesh->Bind_Resources();    // 각 메시의 VB/IB를 IA에 세팅
    pMesh->Render();            // 각 메시마다 DrawIndexed()
}
```

여러 메시 = **여러 번의 DrawCall**. 모델의 메시 수만큼 IA 바인딩과 DrawIndexed가 반복된다.

### 10.3 핵심 차이

| | CVIBuffer 기반 | CModel 기반 |
|---|---|---|
| IA 바인딩 | GameObject가 직접 | CModel::Render() 내부에서 자동 |
| DrawCall 수 | 1회 | 메시 수만큼 |
| 텍스처 | CTexture 컴포넌트로 관리 | 현재 미구현 (향후 aiMaterial에서 추출) |
| 셰이더 Pass | Begin 한 번 → 1 Draw | Begin 한 번 → N Draw (같은 Pass) |

## 11. 수업 코드와 내 프로젝트의 차이점

### 11.1 9일차에서 완료된 Assimp 통합

9일차 Light System 적용 이후 Assimp 통합 작업도 이미 완료된 상태:

| 항목 | 상태 |
|------|------|
| Engine_Defines.h — Assimp include + `using namespace Assimp` | ✅ 적용됨 |
| Engine.vcxproj — assimp lib 링크 (Debug/Release) | ✅ 적용됨 |
| Model.h / Model.cpp — 9일차 골격 | ✅ 생성됨 |
| Assimp 헤더/lib/dll 배치 | ✅ 완료 |

10일차에서는 기존 Model.h/Model.cpp를 **10일차 버전으로 확장**하고, CMesh를 새로 추가하면 된다.

### 11.2 입력 시스템 차이 (이전과 동일)

| | 수업 코드 | 내 프로젝트 |
|---|---|---|
| 방식 | DirectInput8 | Raw Input |
| 열거형 | `DIMB`, `DIMM` | `MOUSEBTN`, `MOUSEAXIS` |
| Engine_Defines.h | `#include <dinput.h>` 있음 | 불필요 (추가하지 않음) |

### 11.3 Transform 시스템 차이 (이전과 동일)

| | 수업 코드 | 내 프로젝트 |
|---|---|---|
| CTransform | `final` (단일 클래스) | `abstract` + CTransform_3D/2D |
| TRANSFORMTYPE | 없음 | GAMEOBJECT_DESC에 포함 |

CMonster는 3D 오브젝트이므로 내 프로젝트에서는 기본값인 `TRANSFORM_3D`가 자동 적용된다. 별도 처리 불필요.

### 11.4 레이어 이름 차이

| | 수업 코드 | 내 프로젝트 |
|---|---|---|
| Terrain 레이어 | `Layer_BackGround` | `Layer_Terrain` |
| 메서드 이름 | `Ready_Layer_BackGround()` | `Ready_Layer_Terrain()` |

Monster 레이어(`Layer_Monster`)는 새로 추가하므로 차이 없음.
