  2. 새로운 Enum: D3DTS

  Engine_Enum.h에 D3DTS enum이 추가되었습니다.

  enum class D3DTS { VIEW, PROJ, END };

  용도: CUIObject에서 View/Projection 행렬을 배열로 관리할 때 인덱싱에 사용합니다. UI
  오브젝트는 3D 카메라의 View/Proj이 아닌 자체 행렬(항등 View + 직교 Proj)을
  사용하므로, 이를 별도 배열에 저장합니다.

  3. 신규 Engine 클래스: CUIObject (UI 오브젝트 베이스)

  3.1 개념 - 왜 UI 전용 베이스가 필요한가?

  3D 월드의 오브젝트와 2D UI는 근본적으로 다른 좌표계를 사용합니다:

  ┌───────────┬─────────────────────────┬────────────────────────────────────┐
  │           │       3D 오브젝트       │            UI 오브젝트             │
  ├───────────┼─────────────────────────┼────────────────────────────────────┤
  │ 좌표계    │ 월드 좌표 (3D)          │ 스크린 좌표 (2D, 픽셀 단위)        │
  ├───────────┼─────────────────────────┼────────────────────────────────────┤
  │ View 행렬 │ 카메라 기반             │ 항등 행렬 (카메라 무관)            │
  ├───────────┼─────────────────────────┼────────────────────────────────────┤
  │ Proj 행렬 │ 원근 투영 (Perspective) │ 직교 투영 (Orthographic)           │
  ├───────────┼─────────────────────────┼────────────────────────────────────┤
  │ 위치 지정 │ 월드 XYZ                │ 화면 중심좌표 (fCenterX, fCenterY) │
  ├───────────┼─────────────────────────┼────────────────────────────────────┤
  │ 크기 지정 │ 스케일 팩터             │ 픽셀 단위 (fSizeX, fSizeY)         │
  └───────────┴─────────────────────────┴────────────────────────────────────┘

  3.2 클래스 구조

  CBase
   └── CGameObject (m_pTransformCom 자동 생성)
        └── CUIObject (abstract) ← [NEW] UI 전용 베이스
             └── CBackGround (Client)  ← 상속 변경됨

  디스크립터 상속 체인: TRANSFORM_DESC → GAMEOBJECT_DESC → UIOBJECT_DESC →

  3.4 멤버 변수

  _float    m_fSizeX{}, m_fSizeY{};        // UI 크기 (픽셀)

  3.5 Initialize - 핵심 초기화 로직

  HRESULT CUIObject::Initialize(void* pArg)
  {
      if (nullptr == pArg)
          return E_FAIL;

      auto pDesc = static_cast<UIOBJECT_DESC*>(pArg);

      // 1) DESC에서 UI 속성 추출
      m_fCenterX = pDesc->fCenterX;
      m_fCenterY = pDesc->fCenterY;
      m_fSizeX = pDesc->fSizeX;
      m_fSizeY = pDesc->fSizeY;

      // 2) 부모(CGameObject) Initialize → CTransform 생성
      if (FAILED(__super::Initialize(pArg)))
          return E_FAIL;

      // 3) 뷰포트 크기 취득
      _uint              iNumViewport = { 1 };
      D3D11_VIEWPORT     ViewportDesc = {};
      m_pContext->RSGetViewports(&iNumViewport, &ViewportDesc);
      m_fViewWidth = ViewportDesc.Width;
      m_fViewHeight = ViewportDesc.Height;

      // 4) 월드 행렬 세팅 (스크린 좌표 → DirectX 좌표 변환)
      Update_UIState();

      // 5) View 행렬 = 항등 행렬 (카메라 불필요)
      XMStoreFloat4x4(&m_TransformMatrices[ETOUI(D3DTS::VIEW)], XMMatrixIdentity());

      // 6) Projection 행렬 = 직교 투영 (Orthographic)
      XMStoreFloat4x4(&m_TransformMatrices[ETOUI(D3DTS::PROJ)],
          XMMatrixOrthographicLH(ViewportDesc.Width, ViewportDesc.Height, 0.f, 1.f));

      return S_OK;
  }

  3.6 핵심 개념 - 직교 투영 (Orthographic Projection)

  XMMatrixOrthographicLH(Width, Height, NearZ, FarZ)

  - 원근 투영(Perspective): 멀리 있는 물체가 작아짐 → 3D 월드용
  - 직교 투영(Orthographic): 거리에 관계없이 동일 크기 → 2D UI용
  - Width/Height에 뷰포트 크기(1280×720)를 넣으면, 1픽셀 = 1단위가 됨
  - NearZ=0, FarZ=1 → UI는 깊이가 필요 없으므로 최소 범위

  3.7 Update_UIState - 스크린 좌표 → DirectX 좌표 변환

  void CUIObject::Update_UIState()
  {
      // 스케일 = 픽셀 크기 그대로
      m_pTransformCom->Set_Scale(m_fSizeX, m_fSizeY, 1.f);

      // 위치 변환: 스크린 좌표 → DirectX 좌표
      m_pTransformCom->Set_State(STATE::POSITION,
          XMVectorSet(
              m_fCenterX - m_fViewWidth * 0.5f,      // X: 좌측 오프셋
             -m_fCenterY + m_fViewHeight * 0.5f,      // Y: 상하 반전 + 상단 오프셋
              0.f,                                     // Z: 0 (2D)
              1.f                                      // W: 동차좌표
          ));
  }

  왜 이런 변환이 필요한가?

  스크린 좌표계:              DirectX 직교 좌표계:
  (0,0)────────→ X           Y ↑
    │                          │
    │   (CenterX, CenterY)    │    (0,0)
    │                          │──────────→ X
    ↓ Y                        │
                                ↓

  스크린: 좌상단이 원점, Y 아래로 증가
  DirectX: 화면 중앙이 원점, Y 위로 증가

  변환 공식:
  - DirectX_X = ScreenX - ViewWidth / 2
  - DirectX_Y = -ScreenY + ViewHeight / 2

  예시 (1280×720 화면):
  - 화면 중앙 (640, 360) → DirectX (0, 0)
  - 좌상단 (0, 0) → DirectX (-640, 360)
  - 우하단 (1280, 720) → DirectX (640, -360)

  3.8 Bind_ShaderResource - UI 전용 행렬 바인딩

  HRESULT CUIObject::Bind_ShaderResource(CShader* pShader, const _char* pConstantName,
  D3DTS eType)
  {
      return pShader->Bind_Matrix(pConstantName, &m_TransformMatrices[ETOUI(eType)]);
  }

  UI용 View/Proj 행렬을 셰이더에 전달하는 헬퍼 함수입니다. 자식 클래스에서
  __super::Bind_ShaderResource()로 호출합니다.

  7. Add_Component 시스템

  7.1 이전 방식 (4일차) - 수동 클론

  m_pShaderCom = dynamic_cast<CShader*>(
      m_pGameInstance->Clone_Prototype(PROTOTYPE::COMPONENT, level, tag));
  if (nullptr == m_pShaderCom) return E_FAIL;

  - 클론한 컴포넌트를 멤버 변수에만 저장
  - m_Components map에 등록하지 않음
  - 해제 시 개별적으로 Safe_Release 필요

  7.2 새로운 방식 (5일차) - CGameObject::Add_Component

  HRESULT CGameObject::Add_Component(_uint iPrototypeLevelIndex,
      const _wstring& strPrototypeTag,
      const _wstring& strComponentTag,
      CComponent** ppOut, void* pArg)
  {
      // 1) 중복 검사
      if (nullptr != Find_Component(strComponentTag))
          return E_FAIL;

      // 2) Prototype_Manager에서 클론
      CComponent* pComponent = dynamic_cast<CComponent*>(
          m_pGameInstance->Clone_Prototype(PROTOTYPE::COMPONENT,
              iPrototypeLevelIndex, strPrototypeTag, pArg));
      if (nullptr == pComponent)
          return E_FAIL;

      // 3) m_Components map에 등록
      m_Components.emplace(strComponentTag, pComponent);

      // 4) 호출자의 포인터에도 저장
      *ppOut = pComponent;

      // 5) 추가 AddRef (m_Components + 호출자 = 2곳에서 참조)
      Safe_AddRef(pComponent);

      return S_OK;
  }

  장점:
  - 컴포넌트가 m_Components map에 자동 등록됨
  - CGameObject::Free()에서 map 순회하며 일괄 해제
  - Find_Component()로 태그 기반 검색 가능
  - 중복 등록 방지

  참조 카운트 흐름:
  Clone() → RefCnt = 1
  m_Components.emplace() → (map이 포인터만 저장, AddRef 없음)
  Safe_AddRef(pComponent) → RefCnt = 2  (m_Components용 1 + ppOut용 1)

  해제 시:
  CGameObject::Free() → m_Components 순회 Safe_Release → RefCnt -1
  자식 클래스 Free() → Safe_Release(m_pShaderCom) → RefCnt -1 → 최종 해제

  9. 전체 흐름 요약

  렌더링 순서 (CRenderer::Draw)

  1. Render_Priority()   → 배경 등 (가장 먼저, 뒤에 깔림)
  2. Render_NonBlend()   → 불투명 3D 오브젝트 (Terrain 등)
  3. Render_Blend()      → 반투명 오브젝트
  4. Render_UI()         → UI 오브젝트 (가장 나중, 맨 위에 표시) ← BackGround가 여기로
  이동

  UI vs 3D 오브젝트 비교

  CBackGround (UI):
    상속: CUIObject → 직교 투영, 스크린 좌표 → DirectX 좌표 자동 변환
    렌더 그룹: RENDERID::UI
