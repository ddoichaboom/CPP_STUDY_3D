# 8개월차 7일차 - Camera 시스템 완성

## 개요

6일차에서 만든 CCamera(abstract)를 실제로 동작시키기 위한 전체 파이프라인을 구축한다.
- **CPipeLine**: View/Proj 행렬을 엔진 전역에서 공유하는 중앙 저장소
- **CInput_Device**: Raw Input 기반 키보드/마우스 입력 시스템 (현업 방식, DirectInput8 대체)
- **CCamera_Free**: WASD + 마우스로 자유롭게 이동하는 구체 카메라 (Client측)
- **Terrain 개선**: 하드코딩된 View/Proj를 PipeLine에서 가져오도록 변경

> **수업 원본과의 차이:** 수업에서는 DirectInput8을 사용하지만, 본 문서는 현업/상용 엔진(Unreal, Unity)에서
> 표준으로 사용하는 **Raw Input**(키보드/마우스) + **XInput**(게임패드, 향후 확장) 방식으로 구현한다.
> DirectInput8은 Microsoft가 레거시로 분류하였으며, 현업에서는 특수 입력장치 외에는 사용하지 않는다.

## 1. 이론: 렌더링 파이프라인에서 카메라의 역할

### View 행렬과 카메라의 관계

카메라의 Transform(WorldMatrix)은 "카메라가 월드 공간에서 어디에, 어떤 방향으로 있는지"를 나타낸다.
하지만 셰이더에 전달하는 **View 행렬**은 이 WorldMatrix의 **역행렬**이다.

```
View Matrix = Inverse(Camera WorldMatrix)
```

**이유:** View 변환은 "모든 오브젝트를 카메라 기준 좌표계로 변환"하는 것.
카메라가 오른쪽으로 10 이동했다면, 나머지 세상은 카메라 기준으로 왼쪽으로 10 이동한 것과 같다.

### Projection 행렬

원근 투영(Perspective Projection)은 3D 공간을 2D 화면에 매핑한다.

```
Proj Matrix = XMMatrixPerspectiveFovLH(fFovy, fAspect, fNear, fFar)
```

- **fFovy**: 수직 시야각 (Field of View Y, 라디안)
- **fAspect**: 화면 비율 (Width / Height). 뷰포트에서 자동 계산
- **fNear/fFar**: 근평면/원평면. 이 범위 밖의 오브젝트는 클리핑

### PipeLine의 역할

카메라가 매 프레임 View/Proj를 계산해서 **CPipeLine에 저장**하면,
모든 GameObject가 렌더링 시 PipeLine에서 꺼내 쓴다.
→ 하드코딩 제거, 카메라와 오브젝트 간 결합도 제거

### 업데이트 순서 (중요)

```
Input_Device->Update()          // (1) 누적된 Raw Input 데이터를 프레임 데이터로 복사
Object_Manager->Priority_Update // (2) 카메라가 여기서 이동 처리
Object_Manager->Update          // (3) 카메라가 여기서 PipeLine에 View/Proj 세팅
PipeLine->Update()              // (4) 역행렬 계산 + 카메라 위치 추출
Object_Manager->Late_Update     // (5) 오브젝트들이 렌더 그룹에 등록
```

카메라의 `Update()`에서 PipeLine에 행렬을 세팅한 뒤, `PipeLine::Update()`가 역행렬을 계산한다.
이후 `Late_Update`에서 오브젝트들이 렌더링 등록할 때 PipeLine의 행렬을 사용한다.

## 2. Engine_Enum.h - 입력 관련 enum 추가

마우스 버튼/이동 축을 위한 enum 2개 추가.
DirectInput의 `DIMB/DIMM` 네이밍 대신 **현업 스타일 네이밍**을 사용한다.

```cpp
// 기존 enum들 뒤에 추가

// 마우스 버튼 인덱스
enum class MOUSEBTN { LBUTTON, RBUTTON, MBUTTON, END };

// 마우스 이동 축
enum class MOUSEAXIS { X, Y, WHEEL, END };
```

- `MOUSEBTN`: 마우스 버튼. `LBUTTON`=좌, `RBUTTON`=우, `MBUTTON`=휠(중간) 클릭
- `MOUSEAXIS`: 마우스 이동 축. `X`=수평, `Y`=수직, `WHEEL`=휠 스크롤

> **수업 원본과의 차이:** 수업에서는 `DIMB`(DirectInput Mouse Button), `DIMM`(DirectInput Mouse Move)
> 이라는 DirectInput 종속적 이름을 사용하지만, 본 문서는 입력 백엔드에 무관한 범용 이름을 사용한다.

## 4. Engine_Defines.h - DirectInput 제거

### 4.1 변경 내용

DirectInput8 관련 코드를 **전부 제거**한다.
키 입력은 `<Windows.h>`에 이미 정의된 **Virtual Key Code (`VK_*`)** 를 사용하므로,
별도의 매크로 정의가 전혀 필요 없다.

### 4.2 코드 변경

```cpp
// ===== 삭제 =====
// #define DIRECTINPUT_VERSION 0x0800
// #include <dinput.h>

// ===== 추가할 것: 없음 =====
// VK_SPACE, VK_RETURN 등 특수 키는 <Windows.h>에 이미 정의되어 있음
// 알파벳 키(A~Z)는 VK_ 매크로가 없으므로 문자 리터럴('W', 'A' 등) 사용
// <d3d11.h>가 <Windows.h>를 include하므로 자동으로 사용 가능
```

### 4.3 링커 종속성 변경

- **제거**: `dinput8.lib`, `dxguid.lib` (Engine 프로젝트 → 링커 → 입력 → 추가 종속성)
- **추가 없음**: Raw Input은 Win32 API 기본 제공이므로 별도 라이브러리 불필요

### 4.4 Virtual Key Code vs 스캔코드

```
키보드 하드웨어 → 스캔코드(MakeCode) → OS가 Virtual Key로 변환
```

| | 스캔코드 (MakeCode) | Virtual Key Code (VKey) |
|---|---|---|
| 예시 | W키 = `0x11` | W키 = `'W'` (`0x57`) |
| 정의 위치 | `dinput.h` (DIK_W 등) | `<Windows.h>` (VK_SPACE 등), 알파벳은 문자 리터럴 |
| 특징 | 키보드 레이아웃 무관 (물리 위치) | OS 레이아웃 의존 (논리 키) |
| 현업 사용 | DirectInput (레거시) | **Raw Input + 현업 표준** |

- **알파벳 키의 Virtual Key Code는 ASCII 값과 동일**: `'W'` = `0x57`
- 알파벳(A~Z), 숫자(0~9)는 VK_ 매크로가 없으므로 문자 리터럴 사용
- 게임에서 WASD는 물리 위치 기반이 유리하지만, VK 방식이 압도적으로 간결하고 현업 표준

### 4.5 자주 쓰는 VK 코드 참고표

| 용도 | VK 코드 | 값 |
|------|---------|-----|
| W/A/S/D | `'W'` / `'A'` / `'S'` / `'D'` | 0x57 / 0x41 / 0x53 / 0x44 |
| Space | `VK_SPACE` | 0x20 |
| Enter | `VK_RETURN` | 0x0D |
| Escape | `VK_ESCAPE` | 0x1B |
| Shift (좌/우) | `VK_LSHIFT` / `VK_RSHIFT` | 0xA0 / 0xA1 |
| Ctrl (좌/우) | `VK_LCONTROL` / `VK_RCONTROL` | 0xA2 / 0xA3 |
| Alt (좌/우) | `VK_LMENU` / `VK_RMENU` | 0xA4 / 0xA5 |
| Tab | `VK_TAB` | 0x09 |
| 방향키 | `VK_UP` / `VK_DOWN` / `VK_LEFT` / `VK_RIGHT` | 0x26~0x28, 0x25 |
| F1~F12 | `VK_F1` ~ `VK_F12` | 0x70 ~ 0x7B |
| 숫자 0~9 | `'0'` ~ `'9'` | 0x30 ~ 0x39 |

> **참고:** 알파벳과 숫자는 VK 매크로 대신 문자 리터럴(`'W'`, `'0'`)도 사용 가능하다.
> 단, 가독성을 위해 특수 키는 반드시 VK 매크로를 사용한다.

## 5. CPipeLine - View/Proj 행렬 중앙 저장소 (NEW)

### 5.1 이론: 왜 PipeLine이 필요한가

- 카메라가 View/Proj 행렬을 계산
- 모든 오브젝트가 렌더링 시 이 행렬을 사용
- **PipeLine**: 카메라가 세팅하고, 오브젝트가 읽는 **중앙 저장소**

### 5.2 PipeLine.h

```cpp
#pragma once

#include "Base.h"

NS_BEGIN(Engine)

class CPipeLine final : public CBase
{
private:
    CPipeLine();
    virtual ~CPipeLine() = default;

public:
    // View/Proj 행렬 가져오기
    const _float4x4* Get_Transform(D3DTS eState) const {
        return &m_TransformStateMatrices[ETOUI(eState)];
    }

    // View/Proj 역행렬 가져오기
    const _float4x4* Get_Transform_Inverse(D3DTS eState) const {
        return &m_TransformStateInverseMatrices[ETOUI(eState)];
    }

    // 카메라 월드 위치 가져오기
    const _float4* Get_CamPosition() const {
        return &m_vCamPosition;
    }

public:
    // 카메라가 View/Proj 행렬을 세팅
    void Set_Transform(D3DTS eState, _fmatrix StateMatrix) {
        XMStoreFloat4x4(&m_TransformStateMatrices[ETOUI(eState)], StateMatrix);
    }

public:
    void Update();

private:
    _float4x4   m_TransformStateMatrices[ETOUI(D3DTS::END)] = {};
    _float4x4   m_TransformStateInverseMatrices[ETOUI(D3DTS::END)] = {};
    _float4     m_vCamPosition = {};

public:
    static CPipeLine* Create();
    virtual void Free() override;
};

NS_END
```

### 5.3 PipeLine.cpp

```cpp
#include "PipeLine.h"

CPipeLine::CPipeLine()
{
    // 항등행렬로 초기화
    for (size_t i = 0; i < ETOUI(D3DTS::END); ++i)
    {
        XMStoreFloat4x4(&m_TransformStateMatrices[i], XMMatrixIdentity());
        XMStoreFloat4x4(&m_TransformStateInverseMatrices[i], XMMatrixIdentity());
    }
}

void CPipeLine::Update()
{
    // 모든 변환 행렬의 역행렬 계산
    for (size_t i = 0; i < ETOUI(D3DTS::END); ++i)
    {
        XMStoreFloat4x4(&m_TransformStateInverseMatrices[i],
            XMMatrixInverse(nullptr, XMLoadFloat4x4(&m_TransformStateMatrices[i])));
    }

    // View 역행렬의 4번째 행 = 카메라 월드 위치
    memcpy(&m_vCamPosition,
        &m_TransformStateInverseMatrices[ETOUI(D3DTS::VIEW)]._41,
        sizeof m_vCamPosition);
}

CPipeLine* CPipeLine::Create()
{
    return new CPipeLine();
}

void CPipeLine::Free()
{
    __super::Free();
}
```

### 5.4 핵심 개념: 카메라 위치 추출

```
View 역행렬 = Camera WorldMatrix
```

`_float4x4`의 4번째 행(`._41, ._42, ._43, ._44`)은 POSITION 행이므로,
View 역행렬의 4번째 행에서 바로 카메라 월드 위치를 `memcpy`로 추출한다.

## 6. CInput_Device - Raw Input 입력 시스템 (NEW)

### 6.1 이론: DirectInput vs Raw Input

| | DirectInput8 (수업 원본) | Raw Input (본 문서) |
|---|---|---|
| 상태 | **레거시** (MS 비권장) | **현행 표준** (Win32 네이티브) |
| 키보드 | 가능 (폴링) | 가능 (이벤트 기반) |
| 마우스 이동량 | 가능 (폴링) | 가능 (**더 정확한 delta**) |
| 키 코드 | DIK_* 스캔코드 (dinput.h 필요) | **VK_* Virtual Key** (Windows.h, 추가 정의 불필요) |
| 데이터 흐름 | `Update()`에서 `GetDeviceState` 폴링 | `WndProc`에서 `WM_INPUT` 수신 → 누적 → `Update()`에서 복사 |
| COM 초기화 | `DirectInput8Create` 필요 | **불필요** |
| 추가 라이브러리 | `dinput8.lib`, `dxguid.lib` | **없음** (Win32 기본) |
| 디바이스 구분 | 불가 | **가능** (마우스 2개 등 개별 식별) |
| Xbox 게임패드 | 트리거 L/R 합쳐짐 | XInput 별도 사용 (향후 확장) |

### 6.2 아키텍처: 2단계 데이터 흐름

DirectInput은 `Update()`에서 `GetDeviceState()`로 폴링하는 단순 구조였지만,
Raw Input은 **WndProc 이벤트 → 누적 → 프레임 복사** 2단계 구조다.

```
WndProc (WM_INPUT)                      Update() (매 프레임)
┌──────────────────┐                    ┌──────────────────┐
│ Process_Input()  │                    │ 누적 버퍼 →      │
│ → static 누적    │  ─── 매 프레임 ──→  │   프레임 데이터   │
│   버퍼에 저장    │                    │ 마우스 누적 초기화│
└──────────────────┘                    └──────────────────┘
 - 키: VKey 기준 눌림 0x80 / 떼짐 0x00  - m_byKeyState[256]
 - 마우스 이동: 누적(+=)                 - m_lMouseDelta[3]
 - 마우스 버튼: 최신 상태 덮어쓰기       - m_byMouseBtnState[3]
```

**왜 2단계인가:** `WM_INPUT` 메시지는 한 프레임에 여러 번 발생할 수 있다.
마우스 이동은 누적해야 하고, 키/버튼 상태는 최종 상태를 유지해야 한다.
`Update()`에서 프레임 데이터로 복사한 뒤 마우스 누적을 초기화한다.

### 6.3 Input_Device.h

```cpp
#pragma once

#include "Base.h"

NS_BEGIN(Engine)

class CInput_Device final : public CBase
{
private:
    CInput_Device();
    virtual ~CInput_Device() = default;

public:
    // 키보드 특정 키 상태. 'W' 등 Virtual Key Code 사용, 0x80 비트로 눌림 판별
    _byte Get_KeyState(_ubyte byKeyID) {
        return m_byKeyState[byKeyID];
    }

    // 마우스 버튼 상태 (MOUSEBTN::LBUTTON 등)
    _byte Get_MouseBtnState(MOUSEBTN eBtn) {
        return m_byMouseBtnState[ETOUI(eBtn)];
    }

    // 마우스 이동량 (MOUSEAXIS::X, Y, WHEEL)
    _long Get_MouseDelta(MOUSEAXIS eAxis) {
        return m_lMouseDelta[ETOUI(eAxis)];
    }

public:
    HRESULT Initialize(HWND hWnd);
    void    Update();

    // WndProc에서 WM_INPUT 발생 시 호출하는 정적 콜백
    static void Process_Input(LPARAM lParam);

private:
    // === 프레임 데이터 (Update에서 복사됨, 게임 로직이 읽음) ===
    _byte   m_byKeyState[256] = {};
    _byte   m_byMouseBtnState[ETOUI(MOUSEBTN::END)] = {};
    _long   m_lMouseDelta[ETOUI(MOUSEAXIS::END)] = {};

    // === 누적 버퍼 (WndProc에서 기록, Update에서 복사 후 초기화) ===
    static _byte s_byRawKeyState[256];
    static _byte s_byRawMouseBtn[ETOUI(MOUSEBTN::END)];
    static _long s_lRawMouseAccum[ETOUI(MOUSEAXIS::END)];

public:
    static CInput_Device* Create(HWND hWnd);
    virtual void Free() override;
};

NS_END
```

### 6.4 Input_Device.cpp

```cpp
#include "Input_Device.h"

// static 멤버 초기화
_byte CInput_Device::s_byRawKeyState[256] = {};
_byte CInput_Device::s_byRawMouseBtn[ETOUI(MOUSEBTN::END)] = {};
_long CInput_Device::s_lRawMouseAccum[ETOUI(MOUSEAXIS::END)] = {};

CInput_Device::CInput_Device()
{
}

HRESULT CInput_Device::Initialize(HWND hWnd)
{
    // Raw Input 디바이스 등록
    RAWINPUTDEVICE rid[2]{};

    // 키보드
    rid[0].usUsagePage = 0x01;   // HID_USAGE_PAGE_GENERIC
    rid[0].usUsage     = 0x06;   // HID_USAGE_GENERIC_KEYBOARD
    rid[0].dwFlags     = 0;      // 포커스 있을 때만 수신
    rid[0].hwndTarget  = hWnd;

    // 마우스
    rid[1].usUsagePage = 0x01;   // HID_USAGE_PAGE_GENERIC
    rid[1].usUsage     = 0x02;   // HID_USAGE_GENERIC_MOUSE
    rid[1].dwFlags     = 0;
    rid[1].hwndTarget  = hWnd;

    if (!RegisterRawInputDevices(rid, 2, sizeof(RAWINPUTDEVICE)))
        return E_FAIL;

    return S_OK;
}

// WndProc의 WM_INPUT 케이스에서 호출
// 여러 번 호출될 수 있으므로 마우스 이동은 누적(+=), 키/버튼은 덮어쓰기
void CInput_Device::Process_Input(LPARAM lParam)
{
    _uint dwSize = 0;
    GetRawInputData(reinterpret_cast<HRAWINPUT>(lParam), RID_INPUT,
        nullptr, &dwSize, sizeof(RAWINPUTHEADER));

    if (0 == dwSize)
        return;

    _ubyte* lpb = new _ubyte[dwSize];
    if (GetRawInputData(reinterpret_cast<HRAWINPUT>(lParam), RID_INPUT,
        lpb, &dwSize, sizeof(RAWINPUTHEADER)) != dwSize)
    {
        Safe_Delete_Array(lpb);
        return;
    }

    RAWINPUT* raw = reinterpret_cast<RAWINPUT*>(lpb);

    if (raw->header.dwType == RIM_TYPEKEYBOARD)
    {
        // VKey = Virtual Key Code ('W' = 0x57 등, 알파벳은 ASCII 값 그대로)
        _ushort vKey = raw->data.keyboard.VKey;

        if (vKey < 256)
        {
            if (raw->data.keyboard.Flags & RI_KEY_BREAK)
                s_byRawKeyState[vKey] = 0x00;   // 키 떼짐
            else
                s_byRawKeyState[vKey] = 0x80;   // 키 눌림
        }
    }
    else if (raw->header.dwType == RIM_TYPEMOUSE)
    {
        // 마우스 이동량 누적 (한 프레임에 WM_INPUT 여러 번 올 수 있음)
        s_lRawMouseAccum[ETOUI(MOUSEAXIS::X)] += raw->data.mouse.lLastX;
        s_lRawMouseAccum[ETOUI(MOUSEAXIS::Y)] += raw->data.mouse.lLastY;

        // 마우스 버튼 상태
        _ushort flags = raw->data.mouse.usButtonFlags;

        if (flags & RI_MOUSE_LEFT_BUTTON_DOWN)
            s_byRawMouseBtn[ETOUI(MOUSEBTN::LBUTTON)] = 0x80;
        if (flags & RI_MOUSE_LEFT_BUTTON_UP)
            s_byRawMouseBtn[ETOUI(MOUSEBTN::LBUTTON)] = 0x00;

        if (flags & RI_MOUSE_RIGHT_BUTTON_DOWN)
            s_byRawMouseBtn[ETOUI(MOUSEBTN::RBUTTON)] = 0x80;
        if (flags & RI_MOUSE_RIGHT_BUTTON_UP)
            s_byRawMouseBtn[ETOUI(MOUSEBTN::RBUTTON)] = 0x00;

        if (flags & RI_MOUSE_MIDDLE_BUTTON_DOWN)
            s_byRawMouseBtn[ETOUI(MOUSEBTN::MBUTTON)] = 0x80;
        if (flags & RI_MOUSE_MIDDLE_BUTTON_UP)
            s_byRawMouseBtn[ETOUI(MOUSEBTN::MBUTTON)] = 0x00;

        // 휠 스크롤
        if (flags & RI_MOUSE_WHEEL)
            s_lRawMouseAccum[ETOUI(MOUSEAXIS::WHEEL)] +=
                static_cast<_short>(raw->data.mouse.usButtonData);
    }

    Safe_Delete_Array(lpb);
}

void CInput_Device::Update()
{
    // 누적된 Raw Input 데이터를 프레임 데이터로 복사
    memcpy(m_byKeyState, s_byRawKeyState, sizeof(m_byKeyState));
    memcpy(m_byMouseBtnState, s_byRawMouseBtn, sizeof(m_byMouseBtnState));
    memcpy(m_lMouseDelta, s_lRawMouseAccum, sizeof(m_lMouseDelta));

    // 마우스 이동량은 프레임당 델타이므로 누적 초기화
    // 키/버튼 상태는 초기화하지 않음 (눌린 채로 유지되어야 함)
    ZeroMemory(s_lRawMouseAccum, sizeof(s_lRawMouseAccum));
}

CInput_Device* CInput_Device::Create(HWND hWnd)
{
    CInput_Device* pInstance = new CInput_Device();

    if (FAILED(pInstance->Initialize(hWnd)))
    {
        MSG_BOX("Failed to Created : CInput_Device");
        Safe_Release(pInstance);
    }

    return pInstance;
}

void CInput_Device::Free()
{
    // Raw Input은 COM 객체가 없으므로 별도 해제 불필요
}
```

### 6.5 핵심 개념

**키 상태 판별**
```cpp
if (m_pGameInstance->Get_KeyState('W') & 0x80)
```
`'W'`는 ASCII 값 0x57이며 Virtual Key Code와 동일. 알파벳은 VK_ 매크로가 없으므로 문자 리터럴 사용.

**마우스 이동량**

Raw Input의 `RAWINPUT::data.mouse.lLastX/lLastY`는 프레임 간 이동 델타(픽셀 단위).
DirectInput의 `DIMOUSESTATE::lX/lY`와 동일한 의미이므로 카메라 회전 로직 변경 불필요.

**Static 버퍼를 쓰는 이유**

`Process_Input()`은 `WndProc`에서 호출되는 정적 함수이므로, 인스턴스 멤버에 직접 접근할 수 없다.
정적 버퍼에 누적한 뒤 `Update()`에서 인스턴스 멤버로 복사하는 구조.

> **참고**: 더 깔끔한 설계를 원하면 `CInput_Device*` 싱글턴 포인터를 정적 멤버로 갖고
> `Process_Input`에서 인스턴스 멤버에 직접 쓰는 방법도 있다. 현재 구조는 단순성을 우선함.

**비교: DirectInput 폴링 vs Raw Input 이벤트**

```
[DirectInput - 수업 원본]
Update() {
    m_pKeyBoard->GetDeviceState(256, m_byKeyState);    // OS에 "지금 키 상태 알려줘" 폴링
    m_pMouse->GetDeviceState(sizeof(DIMOUSESTATE), &m_tMouseState);
}

[Raw Input - 본 문서]
WndProc → Process_Input()  // OS가 "입력 왔어" 이벤트 푸시 → 누적
Update() {
    memcpy(프레임 데이터, 누적 버퍼);  // 누적된 것을 복사
    ZeroMemory(마우스 누적);            // 다음 프레임 준비
}
```

```cpp
void CCamera_Free::Priority_Update(_float fTimeDelta)
{
    // --- WASD 이동 ---
    // 알파벳 키는 문자 리터럴('W' 등)이 곧 Virtual Key Code

    if (m_pGameInstance->Get_KeyState('W') & 0x80)
        static_cast<CTransform_3D*>(m_pTransformCom)->Go_Straight(fTimeDelta);

    if (m_pGameInstance->Get_KeyState('S') & 0x80)
        static_cast<CTransform_3D*>(m_pTransformCom)->Go_Backward(fTimeDelta);

    if (m_pGameInstance->Get_KeyState('A') & 0x80)

    if (MouseMove = m_pGameInstance->Get_MouseDelta(MOUSEAXIS::X))

        static_cast<CTransform_3D*>(m_pTransformCom)->Turn(
            XMVectorSet(0.f, 1.f, 0.f, 0.f),
            MouseMove * m_fMouseSensor * fTimeDelta);
    }

    // 수직 이동 → Right축 기준 회전 (Pitch)
    if (MouseMove = m_pGameInstance->Get_MouseDelta(MOUSEAXIS::Y))
    {
        static_cast<CTransform_3D*>(m_pTransformCom)->Turn(
            m_pTransformCom->Get_State(STATE::RIGHT),
            MouseMove * m_fMouseSensor * fTimeDelta);
    }

    // 부모(CCamera)의 Priority_Update 호출
    __super::Priority_Update(fTimeDelta);
}
```

### 9.4 핵심 개념: 마우스 회전

**Yaw (수평 회전):** 마우스 X 이동 → 월드 Y축(0,1,0) 기준 Turn
```
마우스 오른쪽 이동 → 카메라 오른쪽 회전
```


**fMouseSensor**: 마우스 감도. 이동량에 곱해서 회전 속도 조절.
Turn 내부에서 `m_fRotationPerSec`가 곱해지므로, 최종 회전량 = `MouseMove * fMouseSensor * fTimeDelta * m_fRotationPerSec`

### 10.3 Client.cpp - WndProc에 WM_INPUT 처리 추가 (Raw Input 핵심)

DirectInput에서는 WndProc 수정이 불필요했지만, **Raw Input은 WndProc에서 메시지를 받아야 한다.**
이것이 DirectInput → Raw Input 전환 시 유일한 추가 작업이다.

```cpp
// #include "Input_Device.h" 불필요 — CGameInstance를 경유하므로 Engine 내부 클래스 직접 참조 없음

LRESULT CALLBACK WndProc(HWND hWnd, UINT message, WPARAM wParam, LPARAM lParam)
{
    switch (message)
    {
    case WM_INPUT:                              // NEW: Raw Input 메시지 처리
        CGameInstance::Process_RawInput(lParam);
        break;

    case WM_DESTROY:
        PostQuitMessage(0);
        break;

    default:
        return DefWindowProc(hWnd, message, wParam, lParam);
    }
    return 0;
}
```

> **왜 WndProc인가:** Raw Input은 이벤트 기반. OS가 입력 발생 시 `WM_INPUT` 메시지를 보내므로,
> WndProc에서 받아서 `CGameInstance::Process_RawInput()`으로 전달해야 한다.
> Client는 `CInput_Device`를 직접 알 필요 없이 `CGameInstance` 경유로 처리한다.
> DirectInput은 폴링 기반이라 `Update()`에서 `GetDeviceState()`만 호출하면 됐었다.

## 12. 부록: DirectInput vs Raw Input 비교

### 수업 원본 (DirectInput8) 방식

```
초기화: DirectInput8Create() → CreateDevice(키보드/마우스) → SetDataFormat → SetCooperativeLevel → Acquire
매 프레임: GetDeviceState(256, keyState) / GetDeviceState(sizeof(DIMOUSESTATE), &mouseState)
해제: COM Release (m_pKeyBoard, m_pMouse, m_pInputSDK)
필요 라이브러리: dinput8.lib, dxguid.lib
키 코드: DIK_* 스캔코드 (dinput.h 정의)
```

### 본 문서 (Raw Input) 방식

```
초기화: RegisterRawInputDevices(키보드 + 마우스)
WndProc: WM_INPUT → GetRawInputData() → Process_Input()에서 static 버퍼에 누적
매 프레임: Update()에서 static 버퍼 → 인스턴스 멤버로 복사, 마우스 누적 초기화
해제: 없음 (Win32 네이티브)
필요 라이브러리: 없음
키 코드: VK_* Virtual Key Code (<Windows.h> 정의, 추가 작업 불필요)
```

### 외부 인터페이스 비교

| 메서드 | 수업 원본 | 본 문서 | 반환값 |
|--------|----------|---------|--------|
| 키보드 | `Get_DIKeyState(DIK_W)` | `Get_KeyState('W')` | `0x80` = 눌림 |
| 마우스 버튼 | `Get_DIMouseState(DIMB::LBUTTON)` | `Get_MouseBtnState(MOUSEBTN::LBUTTON)` | `0x80` = 눌림 |
| 마우스 이동 | `Get_DIMouseMove(DIMM::X)` | `Get_MouseDelta(MOUSEAXIS::X)` | 이동 델타(픽셀) |

### 향후 확장: XInput (게임패드)

현재는 키보드+마우스만 구현했지만, Xbox 게임패드 지원이 필요하면
CInput_Device에 XInput 폴링을 추가하면 된다:

```cpp
// Input_Device.h에 추가
#include <Xinput.h>
#pragma comment(lib, "xinput.lib")

// 게임패드 관련 멤버
XINPUT_STATE    m_GamepadState = {};
_bool           m_bGamepadConnected = false;

// 게임패드 인터페이스
_bool Is_GamepadConnected() const { return m_bGamepadConnected; }
_bool Is_GamepadBtnDown(WORD wBtn) const;     // XINPUT_GAMEPAD_A 등
_short Get_GamepadThumbL_X() const;           // 왼쪽 스틱 X (-32768~32767)
_short Get_GamepadThumbL_Y() const;           // 왼쪽 스틱 Y
_byte Get_GamepadTriggerL() const;            // 왼쪽 트리거 (0~255)
_byte Get_GamepadTriggerR() const;            // 오른쪽 트리거 (0~255)
void Set_GamepadVibration(_ushort wLeft, _ushort wRight);  // 진동

// Update()에 추가
DWORD result = XInputGetState(0, &m_GamepadState);
m_bGamepadConnected = (result == ERROR_SUCCESS);
```

XInput은 Raw Input과 달리 **폴링 방식**이므로 `Update()`에서 호출하면 된다.
키보드/마우스(Raw Input, 이벤트 기반)와 게임패드(XInput, 폴링 기반)를
하나의 `CInput_Device::Update()`에서 통합 처리하는 것이 현업 패턴이다.
